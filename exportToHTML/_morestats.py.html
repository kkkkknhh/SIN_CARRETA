50   def bayes_mvs(data, alpha=0.90):
51       r""" 
52       Bayesian confidence intervals for the mean, var, and std. 
53    
54       Parameters 
55       ---------- 
56       data : array_like 
57           Input data, if multi-dimensional it is flattened to 1-D by `bayes_mvs`. 
58           Requires 2 or more data points. 
59       alpha : float, optional 
60           Probability that the returned confidence interval contains 
61           the true parameter. 
62    
63       Returns 
64       ------- 
65       mean_cntr, var_cntr, std_cntr : tuple 
66           The three results are for the mean, variance and standard deviation, 
67           respectively.  Each result is a tuple of the form:: 
68    
69               (center, (lower, upper)) 
70    
71           with ``center`` the mean of the conditional pdf of the value given the 
72           data, and ``(lower, upper)`` a confidence interval, centered on the 
73           median, containing the estimate to a probability ``alpha``. 
74    
75       See Also 
76       -------- 
77       mvsdist 
78    
79       Notes 
80       ----- 
81       Each tuple of mean, variance, and standard deviation estimates represent 
82       the (center, (lower, upper)) with center the mean of the conditional pdf 
83       of the value given the data and (lower, upper) is a confidence interval 
84       centered on the median, containing the estimate to a probability 
85       ``alpha``. 
86    
87       Converts data to 1-D and assumes all data has the same mean and variance. 
88       Uses Jeffrey's prior for variance and std. 
89    
90       Equivalent to ``tuple((x.mean(), x.interval(alpha)) for x in mvsdist(dat))`` 
91    
92       References 
93       ---------- 
94       T.E. Oliphant, "A Bayesian perspective on estimating mean, variance, and 
95       standard-deviation from data", https://scholarsarchive.byu.edu/facpub/278, 
96       2006. 
97    
98       Examples 
99       -------- 
100      First a basic example to demonstrate the outputs: 
101   
102      >>> from scipy import stats 
103      >>> data = [6, 9, 12, 7, 8, 8, 13] 
104      >>> mean, var, std = stats.bayes_mvs(data) 
105      >>> mean 
106      Mean(statistic=9.0, minmax=(7.103650222612533, 10.896349777387467)) 
107      >>> var 
108      Variance(statistic=10.0, minmax=(3.176724206, 24.45910382)) 
109      >>> std 
110      Std_dev(statistic=2.9724954732045084, 
111              minmax=(1.7823367265645143, 4.945614605014631)) 
112   
113      Now we generate some normally distributed random data, and get estimates of 
114      mean and standard deviation with 95% confidence intervals for those 
115      estimates: 
116   
117      >>> n_samples = 100000 
118      >>> data = stats.norm.rvs(size=n_samples) 
119      >>> res_mean, res_var, res_std = stats.bayes_mvs(data, alpha=0.95) 
120   
121      >>> import matplotlib.pyplot as plt 
122      >>> fig = plt.figure() 
123      >>> ax = fig.add_subplot(111) 
124      >>> ax.hist(data, bins=100, density=True, label='Histogram of data') 
125      >>> ax.vlines(res_mean.statistic, 0, 0.5, colors='r', label='Estimated mean') 
126      >>> ax.axvspan(res_mean.minmax[0],res_mean.minmax[1], facecolor='r', 
127      ...            alpha=0.2, label=r'Estimated mean (95% limits)') 
128      >>> ax.vlines(res_std.statistic, 0, 0.5, colors='g', label='Estimated scale') 
129      >>> ax.axvspan(res_std.minmax[0],res_std.minmax[1], facecolor='g', alpha=0.2, 
130      ...            label=r'Estimated scale (95% limits)') 
131   
132      >>> ax.legend(fontsize=10) 
133      >>> ax.set_xlim([-4, 4]) 
134      >>> ax.set_ylim([0, 0.5]) 
135      >>> plt.show() 
136   
137      """
138      m, v, s = mvsdist(data)
139      if alpha >= 1 or alpha <= 0:
140          raise ValueError(f"0 < alpha < 1 is required, but {alpha=} was given.")
141  
142      m_res = Mean(m.mean(), m.interval(alpha))
143      v_res = Variance(v.mean(), v.interval(alpha))
144      s_res = Std_dev(s.mean(), s.interval(alpha))
145  
146      return m_res, v_res, s_res

149  def mvsdist(data):
150      """ 
151      'Frozen' distributions for mean, variance, and standard deviation of data. 
152   
153      Parameters 
154      ---------- 
155      data : array_like 
156          Input array. Converted to 1-D using ravel. 
157          Requires 2 or more data-points. 
158   
159      Returns 
160      ------- 
161      mdist : "frozen" distribution object 
162          Distribution object representing the mean of the data. 
163      vdist : "frozen" distribution object 
164          Distribution object representing the variance of the data. 
165      sdist : "frozen" distribution object 
166          Distribution object representing the standard deviation of the data. 
167   
168      See Also 
169      -------- 
170      bayes_mvs 
171   
172      Notes 
173      ----- 
174      The return values from ``bayes_mvs(data)`` is equivalent to 
175      ``tuple((x.mean(), x.interval(0.90)) for x in mvsdist(data))``. 
176   
177      In other words, calling ``<dist>.mean()`` and ``<dist>.interval(0.90)`` 
178      on the three distribution objects returned from this function will give 
179      the same results that are returned from `bayes_mvs`. 
180   
181      References 
182      ---------- 
183      T.E. Oliphant, "A Bayesian perspective on estimating mean, variance, and 
184      standard-deviation from data", https://scholarsarchive.byu.edu/facpub/278, 
185      2006. 
186   
187      Examples 
188      -------- 
189      >>> from scipy import stats 
190      >>> data = [6, 9, 12, 7, 8, 8, 13] 
191      >>> mean, var, std = stats.mvsdist(data) 
192   
193      We now have frozen distribution objects "mean", "var" and "std" that we can 
194      examine: 
195   
196      >>> mean.mean() 
197      9.0 
198      >>> mean.interval(0.95) 
199      (6.6120585482655692, 11.387941451734431) 
200      >>> mean.std() 
201      1.1952286093343936 
202   
203      """
204      x = ravel(data)
205      n = len(x)
206      if n < 2:
207          raise ValueError("Need at least 2 data-points.")
208      xbar = x.mean()
209      C = x.var()
210      if n > 1000:  # gaussian approximations for large n
211          mdist = distributions.norm(loc=xbar, scale=math.sqrt(C / n))
212          sdist = distributions.norm(loc=math.sqrt(C), scale=math.sqrt(C / (2. * n)))
213          vdist = distributions.norm(loc=C, scale=math.sqrt(2.0 / n) * C)
214      else:
215          nm1 = n - 1
216          fac = n * C / 2.
217          val = nm1 / 2.
218          mdist = distributions.t(nm1, loc=xbar, scale=math.sqrt(C / nm1))
219          sdist = distributions.gengamma(val, -2, scale=math.sqrt(fac))
220          vdist = distributions.invgamma(val, scale=fac)
221      return mdist, vdist, sdist

227  def kstat(data, n=2, *, axis=None):
228      r""" 
229      Return the `n` th k-statistic ( ``1<=n<=4`` so far). 
230   
231      The `n` th k-statistic ``k_n`` is the unique symmetric unbiased estimator of the 
232      `n` th cumulant :math:`\kappa_n` [1]_ [2]_. 
233   
234      Parameters 
235      ---------- 
236      data : array_like 
237          Input array. 
238      n : int, {1, 2, 3, 4}, optional 
239          Default is equal to 2. 
240      axis : int or None, default: None 
241          If an int, the axis of the input along which to compute the statistic. 
242          The statistic of each axis-slice (e.g. row) of the input will appear 
243          in a corresponding element of the output. If ``None``, the input will 
244          be raveled before computing the statistic. 
245   
246      Returns 
247      ------- 
248      kstat : float 
249          The `n` th k-statistic. 
250   
251      See Also 
252      -------- 
253      kstatvar : Returns an unbiased estimator of the variance of the k-statistic 
254      moment : Returns the n-th central moment about the mean for a sample. 
255   
256      Notes 
257      ----- 
258      For a sample size :math:`n`, the first few k-statistics are given by 
259   
260      .. math:: 
261   
262          k_1 &= \frac{S_1}{n}, \\ 
263          k_2 &= \frac{nS_2 - S_1^2}{n(n-1)}, \\ 
264          k_3 &= \frac{2S_1^3 - 3nS_1S_2 + n^2S_3}{n(n-1)(n-2)}, \\ 
265          k_4 &= \frac{-6S_1^4 + 12nS_1^2S_2 - 3n(n-1)S_2^2 - 4n(n+1)S_1S_3 
266          + n^2(n+1)S_4}{n (n-1)(n-2)(n-3)}, 
267   
268      where 
269   
270      .. math:: 
271   
272          S_r \equiv \sum_{i=1}^n X_i^r, 
273   
274      and :math:`X_i` is the :math:`i` th data point. 
275   
276      References 
277      ---------- 
278      .. [1] http://mathworld.wolfram.com/k-Statistic.html 
279   
280      .. [2] http://mathworld.wolfram.com/Cumulant.html 
281   
282      Examples 
283      -------- 
284      >>> from scipy import stats 
285      >>> from numpy.random import default_rng 
286      >>> rng = default_rng() 
287   
288      As sample size increases, `n`-th moment and `n`-th k-statistic converge to the 
289      same number (although they aren't identical). In the case of the normal 
290      distribution, they converge to zero. 
291   
292      >>> for i in range(2,8): 
293      ...     x = rng.normal(size=10**i) 
294      ...     m, k = stats.moment(x, 3), stats.kstat(x, 3) 
295      ...     print(f"{i=}: {m=:.3g}, {k=:.3g}, {(m-k)=:.3g}") 
296      i=2: m=-0.631, k=-0.651, (m-k)=0.0194  # random 
297      i=3: m=0.0282, k=0.0283, (m-k)=-8.49e-05 
298      i=4: m=-0.0454, k=-0.0454, (m-k)=1.36e-05 
299      i=6: m=7.53e-05, k=7.53e-05, (m-k)=-2.26e-09 
300      i=7: m=0.00166, k=0.00166, (m-k)=-4.99e-09 
301      i=8: m=-2.88e-06 k=-2.88e-06, (m-k)=8.63e-13 
302      """
303      xp = array_namespace(data)
304      data = xp.asarray(data)
305      if n > 4 or n < 1:
306          raise ValueError("k-statistics only supported for 1<=n<=4")
307      n = int(n)
308      if axis is None:
309          data = xp.reshape(data, (-1,))
310          axis = 0
311  
312      N = data.shape[axis]
313  
314      S = [None] + [xp.sum(data**k, axis=axis) for k in range(1, n + 1)]
315      if n == 1:
316          return S[1] * 1.0/N
317      elif n == 2:
318          return (N*S[2] - S[1]**2.0) / (N*(N - 1.0))
319      elif n == 3:
320          return (2*S[1]**3 - 3*N*S[1]*S[2] + N*N*S[3]) / (N*(N - 1.0)*(N - 2.0))
321      elif n == 4:
322          return ((-6*S[1]**4 + 12*N*S[1]**2 * S[2] - 3*N*(N-1.0)*S[2]**2 -
323                   4*N*(N+1)*S[1]*S[3] + N*N*(N+1)*S[4]) /
324                  (N*(N-1.0)*(N-2.0)*(N-3.0)))
325      else:
326          raise ValueError("Should not be here.")

332  def kstatvar(data, n=2, *, axis=None):
333      r"""Return an unbiased estimator of the variance of the k-statistic. 
334   
335      See `kstat` and [1]_ for more details about the k-statistic. 
336   
337      Parameters 
338      ---------- 
339      data : array_like 
340          Input array. 
341      n : int, {1, 2}, optional 
342          Default is equal to 2. 
343      axis : int or None, default: None 
344          If an int, the axis of the input along which to compute the statistic. 
345          The statistic of each axis-slice (e.g. row) of the input will appear 
346          in a corresponding element of the output. If ``None``, the input will 
347          be raveled before computing the statistic. 
348   
349      Returns 
350      ------- 
351      kstatvar : float 
352          The `n` th k-statistic variance. 
353   
354      See Also 
355      -------- 
356      kstat : Returns the n-th k-statistic. 
357      moment : Returns the n-th central moment about the mean for a sample. 
358   
359      Notes 
360      ----- 
361      Unbiased estimators of the variances of the first two k-statistics are given by 
362   
363      .. math:: 
364   
365          \mathrm{var}(k_1) &= \frac{k_2}{n}, \\ 
366          \mathrm{var}(k_2) &= \frac{2k_2^2n + (n-1)k_4}{n(n - 1)}. 
367   
368      References 
369      ---------- 
370      .. [1] http://mathworld.wolfram.com/k-Statistic.html 
371   
372      """  # noqa: E501
373      xp = array_namespace(data)
374      data = xp.asarray(data)
375      if axis is None:
376          data = xp.reshape(data, (-1,))
377          axis = 0
378      N = data.shape[axis]
379  
380      if n == 1:
381          return kstat(data, n=2, axis=axis, _no_deco=True) * 1.0/N
382      elif n == 2:
383          k2 = kstat(data, n=2, axis=axis, _no_deco=True)
384          k4 = kstat(data, n=4, axis=axis, _no_deco=True)
385          return (2*N*k2**2 + (N-1)*k4) / (N*(N+1))
386      else:
387          raise ValueError("Only n=1 or n=2 supported.")

390  def _calc_uniform_order_statistic_medians(n):
391      """Approximations of uniform order statistic medians. 
392   
393      Parameters 
394      ---------- 
395      n : int 
396          Sample size. 
397   
398      Returns 
399      ------- 
400      v : 1d float array 
401          Approximations of the order statistic medians. 
402   
403      References 
404      ---------- 
405      .. [1] James J. Filliben, "The Probability Plot Correlation Coefficient 
406             Test for Normality", Technometrics, Vol. 17, pp. 111-117, 1975. 
407   
408      Examples 
409      -------- 
410      Order statistics of the uniform distribution on the unit interval 
411      are marginally distributed according to beta distributions. 
412      The expectations of these order statistic are evenly spaced across 
413      the interval, but the distributions are skewed in a way that 
414      pushes the medians slightly towards the endpoints of the unit interval: 
415   
416      >>> import numpy as np 
417      >>> n = 4 
418      >>> k = np.arange(1, n+1) 
419      >>> from scipy.stats import beta 
420      >>> a = k 
421      >>> b = n-k+1 
422      >>> beta.mean(a, b) 
423      array([0.2, 0.4, 0.6, 0.8]) 
424      >>> beta.median(a, b) 
425      array([0.15910358, 0.38572757, 0.61427243, 0.84089642]) 
426   
427      The Filliben approximation uses the exact medians of the smallest 
428      and greatest order statistics, and the remaining medians are approximated 
429      by points spread evenly across a sub-interval of the unit interval: 
430   
431      >>> from scipy.stats._morestats import _calc_uniform_order_statistic_medians 
432      >>> _calc_uniform_order_statistic_medians(n) 
433      array([0.15910358, 0.38545246, 0.61454754, 0.84089642]) 
434   
435      This plot shows the skewed distributions of the order statistics 
436      of a sample of size four from a uniform distribution on the unit interval: 
437   
438      >>> import matplotlib.pyplot as plt 
439      >>> x = np.linspace(0.0, 1.0, num=50, endpoint=True) 
440      >>> pdfs = [beta.pdf(x, a[i], b[i]) for i in range(n)] 
441      >>> plt.figure() 
442      >>> plt.plot(x, pdfs[0], x, pdfs[1], x, pdfs[2], x, pdfs[3]) 
443   
444      """
445      v = np.empty(n, dtype=np.float64)
446      v[-1] = 0.5**(1.0 / n)
447      v[0] = 1 - v[-1]
448      i = np.arange(2, n)
449      v[1:-1] = (i - 0.3175) / (n + 0.365)
450      return v

453  def _parse_dist_kw(dist, enforce_subclass=True):
454      """Parse `dist` keyword. 
455   
456      Parameters 
457      ---------- 
458      dist : str or stats.distributions instance. 
459          Several functions take `dist` as a keyword, hence this utility 
460          function. 
461      enforce_subclass : bool, optional 
462          If True (default), `dist` needs to be a 
463          `_distn_infrastructure.rv_generic` instance. 
464          It can sometimes be useful to set this keyword to False, if a function 
465          wants to accept objects that just look somewhat like such an instance 
466          (for example, they have a ``ppf`` method). 
467   
468      """
469      if isinstance(dist, rv_generic):
470          pass
471      elif isinstance(dist, str):
472          try:
473              dist = getattr(distributions, dist)
474          except AttributeError as e:
475              raise ValueError(f"{dist} is not a valid distribution name") from e
476      elif enforce_subclass:
477          msg = ("`dist` should be a stats.distributions instance or a string "
478                 "with the name of such a distribution.")
479          raise ValueError(msg)
480  
481      return dist

503  def probplot(x, sparams=(), dist='norm', fit=True, plot=None, rvalue=False):
504      """ 
505      Calculate quantiles for a probability plot, and optionally show the plot. 
506   
507      Generates a probability plot of sample data against the quantiles of a 
508      specified theoretical distribution (the normal distribution by default). 
509      `probplot` optionally calculates a best-fit line for the data and plots the 
510      results using Matplotlib or a given plot function. 
511   
512      Parameters 
513      ---------- 
514      x : array_like 
515          Sample/response data from which `probplot` creates the plot. 
516      sparams : tuple, optional 
517          Distribution-specific shape parameters (shape parameters plus location 
518          and scale). 
519      dist : str or stats.distributions instance, optional 
520          Distribution or distribution function name. The default is 'norm' for a 
521          normal probability plot.  Objects that look enough like a 
522          stats.distributions instance (i.e. they have a ``ppf`` method) are also 
523          accepted. 
524      fit : bool, optional 
525          Fit a least-squares regression (best-fit) line to the sample data if 
526          True (default). 
527      plot : object, optional 
528          If given, plots the quantiles. 
529          If given and `fit` is True, also plots the least squares fit. 
530          `plot` is an object that has to have methods "plot" and "text". 
531          The `matplotlib.pyplot` module or a Matplotlib Axes object can be used, 
532          or a custom object with the same methods. 
533          Default is None, which means that no plot is created. 
534      rvalue : bool, optional 
535          If `plot` is provided and `fit` is True, setting `rvalue` to True 
536          includes the coefficient of determination on the plot. 
537          Default is False. 
538   
539      Returns 
540      ------- 
541      (osm, osr) : tuple of ndarrays 
542          Tuple of theoretical quantiles (osm, or order statistic medians) and 
543          ordered responses (osr).  `osr` is simply sorted input `x`. 
544          For details on how `osm` is calculated see the Notes section. 
545      (slope, intercept, r) : tuple of floats, optional 
546          Tuple  containing the result of the least-squares fit, if that is 
547          performed by `probplot`. `r` is the square root of the coefficient of 
548          determination.  If ``fit=False`` and ``plot=None``, this tuple is not 
549          returned. 
550   
551      Notes 
552      ----- 
553      Even if `plot` is given, the figure is not shown or saved by `probplot`; 
554      ``plt.show()`` or ``plt.savefig('figname.png')`` should be used after 
555      calling `probplot`. 
556   
557      `probplot` generates a probability plot, which should not be confused with 
558      a Q-Q or a P-P plot.  Statsmodels has more extensive functionality of this 
559      type, see ``statsmodels.api.ProbPlot``. 
560   
561      The formula used for the theoretical quantiles (horizontal axis of the 
562      probability plot) is Filliben's estimate:: 
563   
564          quantiles = dist.ppf(val), for 
565   
566                  0.5**(1/n),                  for i = n 
567            val = (i - 0.3175) / (n + 0.365),  for i = 2, ..., n-1 
568                  1 - 0.5**(1/n),              for i = 1 
569   
570      where ``i`` indicates the i-th ordered value and ``n`` is the total number 
571      of values. 
572   
573      Examples 
574      -------- 
575      >>> import numpy as np 
576      >>> from scipy import stats 
577      >>> import matplotlib.pyplot as plt 
578      >>> nsample = 100 
579      >>> rng = np.random.default_rng() 
580   
581      A t distribution with small degrees of freedom: 
582   
583      >>> ax1 = plt.subplot(221) 
584      >>> x = stats.t.rvs(3, size=nsample, random_state=rng) 
585      >>> res = stats.probplot(x, plot=plt) 
586   
587      A t distribution with larger degrees of freedom: 
588   
589      >>> ax2 = plt.subplot(222) 
590      >>> x = stats.t.rvs(25, size=nsample, random_state=rng) 
591      >>> res = stats.probplot(x, plot=plt) 
592   
593      A mixture of two normal distributions with broadcasting: 
594   
595      >>> ax3 = plt.subplot(223) 
596      >>> x = stats.norm.rvs(loc=[0,5], scale=[1,1.5], 
597      ...                    size=(nsample//2,2), random_state=rng).ravel() 
598      >>> res = stats.probplot(x, plot=plt) 
599   
600      A standard normal distribution: 
601   
602      >>> ax4 = plt.subplot(224) 
603      >>> x = stats.norm.rvs(loc=0, scale=1, size=nsample, random_state=rng) 
604      >>> res = stats.probplot(x, plot=plt) 
605   
606      Produce a new figure with a loggamma distribution, using the ``dist`` and 
607      ``sparams`` keywords: 
608   
609      >>> fig = plt.figure() 
610      >>> ax = fig.add_subplot(111) 
611      >>> x = stats.loggamma.rvs(c=2.5, size=500, random_state=rng) 
612      >>> res = stats.probplot(x, dist=stats.loggamma, sparams=(2.5,), plot=ax) 
613      >>> ax.set_title("Probplot for loggamma dist with shape parameter 2.5") 
614   
615      Show the results with Matplotlib: 
616   
617      >>> plt.show() 
618   
619      """
620      x = np.asarray(x)
621      if x.size == 0:
622          if fit:
623              return (x, x), (np.nan, np.nan, 0.0)
624          else:
625              return x, x
626  
627      osm_uniform = _calc_uniform_order_statistic_medians(len(x))
628      dist = _parse_dist_kw(dist, enforce_subclass=False)
629      if sparams is None:
630          sparams = ()
631      if isscalar(sparams):
632          sparams = (sparams,)
633      if not isinstance(sparams, tuple):
634          sparams = tuple(sparams)
635  
636      osm = dist.ppf(osm_uniform, *sparams)
637      osr = sort(x)
638      if fit:
639          # perform a linear least squares fit.
640          slope, intercept, r, prob, _ = _stats_py.linregress(osm, osr)
641  
642      if plot is not None:
643          plot.plot(osm, osr, 'bo')
644          if fit:
645              plot.plot(osm, slope*osm + intercept, 'r-')
646          _add_axis_labels_title(plot, xlabel='Theoretical quantiles',
647                                 ylabel='Ordered Values',
648                                 title='Probability Plot')
649  
650          # Add R^2 value to the plot as text
651          if fit and rvalue:
652              xmin = amin(osm)
653              xmax = amax(osm)
654              ymin = amin(x)
655              ymax = amax(x)
656              posx = xmin + 0.70 * (xmax - xmin)
657              posy = ymin + 0.01 * (ymax - ymin)
658              plot.text(posx, posy, f"$R^2={r ** 2:1.4f}$")
659  
660      if fit:
661          return (osm, osr), (slope, intercept, r)
662      else:
663          return osm, osr

666  def ppcc_max(x, brack=(0.0, 1.0), dist='tukeylambda'):
667      """Calculate the shape parameter that maximizes the PPCC. 
668   
669      The probability plot correlation coefficient (PPCC) plot can be used 
670      to determine the optimal shape parameter for a one-parameter family 
671      of distributions. ``ppcc_max`` returns the shape parameter that would 
672      maximize the probability plot correlation coefficient for the given 
673      data to a one-parameter family of distributions. 
674   
675      Parameters 
676      ---------- 
677      x : array_like 
678          Input array. 
679      brack : tuple, optional 
680          Triple (a,b,c) where (a<b<c). If bracket consists of two numbers (a, c) 
681          then they are assumed to be a starting interval for a downhill bracket 
682          search (see `scipy.optimize.brent`). 
683      dist : str or stats.distributions instance, optional 
684          Distribution or distribution function name.  Objects that look enough 
685          like a stats.distributions instance (i.e. they have a ``ppf`` method) 
686          are also accepted.  The default is ``'tukeylambda'``. 
687   
688      Returns 
689      ------- 
690      shape_value : float 
691          The shape parameter at which the probability plot correlation 
692          coefficient reaches its max value. 
693   
694      See Also 
695      -------- 
696      ppcc_plot, probplot, boxcox 
697   
698      Notes 
699      ----- 
700      The brack keyword serves as a starting point which is useful in corner 
701      cases. One can use a plot to obtain a rough visual estimate of the location 
702      for the maximum to start the search near it. 
703   
704      References 
705      ---------- 
706      .. [1] J.J. Filliben, "The Probability Plot Correlation Coefficient Test 
707             for Normality", Technometrics, Vol. 17, pp. 111-117, 1975. 
708      .. [2] Engineering Statistics Handbook, NIST/SEMATEC, 
709             https://www.itl.nist.gov/div898/handbook/eda/section3/ppccplot.htm 
710   
711      Examples 
712      -------- 
713      First we generate some random data from a Weibull distribution 
714      with shape parameter 2.5: 
715   
716      >>> import numpy as np 
717      >>> from scipy import stats 
718      >>> import matplotlib.pyplot as plt 
719      >>> rng = np.random.default_rng() 
720      >>> c = 2.5 
721      >>> x = stats.weibull_min.rvs(c, scale=4, size=2000, random_state=rng) 
722   
723      Generate the PPCC plot for this data with the Weibull distribution. 
724   
725      >>> fig, ax = plt.subplots(figsize=(8, 6)) 
726      >>> res = stats.ppcc_plot(x, c/2, 2*c, dist='weibull_min', plot=ax) 
727   
728      We calculate the value where the shape should reach its maximum and a 
729      red line is drawn there. The line should coincide with the highest 
730      point in the PPCC graph. 
731   
732      >>> cmax = stats.ppcc_max(x, brack=(c/2, 2*c), dist='weibull_min') 
733      >>> ax.axvline(cmax, color='r') 
734      >>> plt.show() 
735   
736      """
737      dist = _parse_dist_kw(dist)
738      osm_uniform = _calc_uniform_order_statistic_medians(len(x))
739      osr = sort(x)
740  
741      # this function computes the x-axis values of the probability plot
742      #  and computes a linear regression (including the correlation)
743      #  and returns 1-r so that a minimization function maximizes the
744      #  correlation
745      def tempfunc(shape, mi, yvals, func):
746          xvals = func(mi, shape)
747          r, prob = _stats_py.pearsonr(xvals, yvals)
748          return 1 - r
749  
750      return optimize.brent(tempfunc, brack=brack,
751                            args=(osm_uniform, osr, dist.ppf))

754  def ppcc_plot(x, a, b, dist='tukeylambda', plot=None, N=80):
755      """Calculate and optionally plot probability plot correlation coefficient. 
756   
757      The probability plot correlation coefficient (PPCC) plot can be used to 
758      determine the optimal shape parameter for a one-parameter family of 
759      distributions.  It cannot be used for distributions without shape 
760      parameters 
761      (like the normal distribution) or with multiple shape parameters. 
762   
763      By default a Tukey-Lambda distribution (`stats.tukeylambda`) is used. A 
764      Tukey-Lambda PPCC plot interpolates from long-tailed to short-tailed 
765      distributions via an approximately normal one, and is therefore 
766      particularly useful in practice. 
767   
768      Parameters 
769      ---------- 
770      x : array_like 
771          Input array. 
772      a, b : scalar 
773          Lower and upper bounds of the shape parameter to use. 
774      dist : str or stats.distributions instance, optional 
775          Distribution or distribution function name.  Objects that look enough 
776          like a stats.distributions instance (i.e. they have a ``ppf`` method) 
777          are also accepted.  The default is ``'tukeylambda'``. 
778      plot : object, optional 
779          If given, plots PPCC against the shape parameter. 
780          `plot` is an object that has to have methods "plot" and "text". 
781          The `matplotlib.pyplot` module or a Matplotlib Axes object can be used, 
782          or a custom object with the same methods. 
783          Default is None, which means that no plot is created. 
784      N : int, optional 
785          Number of points on the horizontal axis (equally distributed from 
786          `a` to `b`). 
787   
788      Returns 
789      ------- 
790      svals : ndarray 
791          The shape values for which `ppcc` was calculated. 
792      ppcc : ndarray 
793          The calculated probability plot correlation coefficient values. 
794   
795      See Also 
796      -------- 
797      ppcc_max, probplot, boxcox_normplot, tukeylambda 
798   
799      References 
800      ---------- 
801      J.J. Filliben, "The Probability Plot Correlation Coefficient Test for 
802      Normality", Technometrics, Vol. 17, pp. 111-117, 1975. 
803   
804      Examples 
805      -------- 
806      First we generate some random data from a Weibull distribution 
807      with shape parameter 2.5, and plot the histogram of the data: 
808   
809      >>> import numpy as np 
810      >>> from scipy import stats 
811      >>> import matplotlib.pyplot as plt 
812      >>> rng = np.random.default_rng() 
813      >>> c = 2.5 
814      >>> x = stats.weibull_min.rvs(c, scale=4, size=2000, random_state=rng) 
815   
816      Take a look at the histogram of the data. 
817   
818      >>> fig1, ax = plt.subplots(figsize=(9, 4)) 
819      >>> ax.hist(x, bins=50) 
820      >>> ax.set_title('Histogram of x') 
821      >>> plt.show() 
822   
823      Now we explore this data with a PPCC plot as well as the related 
824      probability plot and Box-Cox normplot.  A red line is drawn where we 
825      expect the PPCC value to be maximal (at the shape parameter ``c`` 
826      used above): 
827   
828      >>> fig2 = plt.figure(figsize=(12, 4)) 
829      >>> ax1 = fig2.add_subplot(1, 3, 1) 
830      >>> ax2 = fig2.add_subplot(1, 3, 2)

835      >>> ax3.axvline(c, color='r') 
836      >>> plt.show() 
837

849          plot.plot(svals, ppcc, 'x')
850          _add_axis_labels_title(plot, xlabel='Shape Values',
851                                 ylabel='Prob Plot Corr. Coef.',
852                                 title=f'({dist}) PPCC Plot')

859      res = special.logsumexp(logx, axis=0) - math.log(logx.shape[0])
860      return res

863  def _log_var(logx, xp):
864      # compute log of variance of x from log(x)
865      logmean = _log_mean(logx)
866      # get complex dtype with component dtypes same as `logx` dtype;
867      # see data-apis/array-api#841
868      dtype = xp.result_type(logx.dtype, xp.complex64)
869      pij = xp.full(logx.shape, pi * 1j, dtype=dtype)
870      logxmu = special.logsumexp(xp.stack((logx, logmean + pij)), axis=0)
871      res = (xp.real(xp.asarray(special.logsumexp(2 * logxmu, axis=0)))
872             - math.log(logx.shape[0]))
873      return res

876  def boxcox_llf(lmb, data):
877      r"""The boxcox log-likelihood function. 
878   
879      Parameters 
880      ---------- 
881      lmb : scalar 
882          Parameter for Box-Cox transformation.  See `boxcox` for details. 
883      data : array_like 
884          Data to calculate Box-Cox log-likelihood for.  If `data` is 
885          multi-dimensional, the log-likelihood is calculated along the first 
886          axis. 
887   
888      Returns 
889      ------- 
890      llf : float or ndarray 
891          Box-Cox log-likelihood of `data` given `lmb`.  A float for 1-D `data`, 
892          an array otherwise. 
893   
894      See Also 
895      -------- 
896      boxcox, probplot, boxcox_normplot, boxcox_normmax 
897   
898      Notes 
899      ----- 
900      The Box-Cox log-likelihood function is defined here as 
901   
902      .. math:: 
903   
904          llf = (\lambda - 1) \sum_i(\log(x_i)) - 
905                N/2 \log(\sum_i (y_i - \bar{y})^2 / N), 
906   
907      where ``y`` is the Box-Cox transformed input data ``x``. 
908   
909      Examples 
910      -------- 
911      >>> import numpy as np 
912      >>> from scipy import stats 
913      >>> import matplotlib.pyplot as plt 
914      >>> from mpl_toolkits.axes_grid1.inset_locator import inset_axes 
915   
916      Generate some random variates and calculate Box-Cox log-likelihood values 
917      for them for a range of ``lmbda`` values: 
918   
919      >>> rng = np.random.default_rng() 
920      >>> x = stats.loggamma.rvs(5, loc=10, size=1000, random_state=rng) 
921      >>> lmbdas = np.linspace(-2, 10) 
922      >>> llf = np.zeros(lmbdas.shape, dtype=float) 
923      >>> for ii, lmbda in enumerate(lmbdas): 
924      ...     llf[ii] = stats.boxcox_llf(lmbda, x) 
925   
926      Also find the optimal lmbda value with `boxcox`: 
927   
928      >>> x_most_normal, lmbda_optimal = stats.boxcox(x) 
929   
930      Plot the log-likelihood as function of lmbda.  Add the optimal lmbda as a 
931      horizontal line to check that that's really the optimum: 
932   
933      >>> fig = plt.figure() 
934      >>> ax = fig.add_subplot(111) 
935      >>> ax.plot(lmbdas, llf, 'b.-') 
936      >>> ax.axhline(stats.boxcox_llf(lmbda_optimal, x), color='r') 
937      >>> ax.set_xlabel('lmbda parameter') 
938      >>> ax.set_ylabel('Box-Cox log-likelihood') 
939   
940      Now add some probability plots to show that where the log-likelihood is 
941      maximized the data transformed with `boxcox` looks closest to normal: 
942   
943      >>> locs = [3, 10, 4]  # 'lower left', 'center', 'lower right' 
944      >>> for lmbda, loc in zip([-1, lmbda_optimal, 9], locs): 
945      ...     xt = stats.boxcox(x, lmbda=lmbda) 
946      ...     (osm, osr), (slope, intercept, r_sq) = stats.probplot(xt) 
947      ...     ax_inset = inset_axes(ax, width="20%", height="20%", loc=loc) 
948      ...     ax_inset.plot(osm, osr, 'c.', osm, slope*osm + intercept, 'k-') 
949      ...     ax_inset.set_xticklabels([]) 
950      ...     ax_inset.set_yticklabels([]) 
951      ...     ax_inset.set_title(r'$\lambda=%1.2f$' % lmbda) 
952   
953      >>> plt.show() 
954   
955      """
956      xp = array_namespace(data)
957      data = xp.asarray(data)
958      N = data.shape[0]
959      if N == 0:
960          return xp.nan
961  
962      dt = data.dtype
963      if xp.isdtype(dt, 'integral'):
964          data = xp.asarray(data, dtype=xp.float64)
965          dt = xp.float64
966  
967      logdata = xp.log(data)
968  
969      # Compute the variance of the transformed data.
970      if lmb == 0:
971          logvar = xp.log(xp.var(logdata, axis=0))
972      else:
973          # Transform without the constant offset 1/lmb.  The offset does
974          # not affect the variance, and the subtraction of the offset can
975          # lead to loss of precision.
976          # Division by lmb can be factored out to enhance numerical stability.
977          logx = lmb * logdata
978          logvar = _log_var(logx, xp) - 2 * math.log(abs(lmb))
979  
980      res = (lmb - 1) * xp.sum(logdata, axis=0) - N/2 * logvar
981      res = xp.astype(res, dt)
982      res = res[()] if res.ndim == 0 else res
983      return res

986  def _boxcox_conf_interval(x, lmax, alpha):
987      # Need to find the lambda for which
988      #  f(x,lmbda) >= f(x,lmax) - 0.5*chi^2_alpha;1
989      fac = 0.5 * distributions.chi2.ppf(1 - alpha, 1)
990      target = boxcox_llf(lmax, x) - fac
991  
992      def rootfunc(lmbda, data, target):
993          return boxcox_llf(lmbda, data) - target
994  
995      # Find positive endpoint of interval in which answer is to be found
996      newlm = lmax + 0.5
997      N = 0
998      while (rootfunc(newlm, x, target) > 0.0) and (N < 500):
999          newlm += 0.1
1000         N += 1
1001 
1002     if N == 500:
1003         raise RuntimeError("Could not find endpoint.")
1004 
1005     lmplus = optimize.brentq(rootfunc, lmax, newlm, args=(x, target))
1006 
1007     # Now find negative interval in the same way
1008     newlm = lmax - 0.5
1009     N = 0
1010     while (rootfunc(newlm, x, target) > 0.0) and (N < 500):
1011         newlm -= 0.1
1012         N += 1
1013 
1014     if N == 500:
1015         raise RuntimeError("Could not find endpoint.")
1016 
1017     lmminus = optimize.brentq(rootfunc, newlm, lmax, args=(x, target))
1018     return lmminus, lmplus

1021 def boxcox(x, lmbda=None, alpha=None, optimizer=None):
1022     r"""Return a dataset transformed by a Box-Cox power transformation. 
1023  
1024     Parameters 
1025     ---------- 
1026     x : ndarray 
1027         Input array to be transformed. 
1028  
1029         If `lmbda` is not None, this is an alias of 
1030         `scipy.special.boxcox`. 
1031         Returns nan if ``x < 0``; returns -inf if ``x == 0 and lmbda < 0``. 
1032  
1033         If `lmbda` is None, array must be positive, 1-dimensional, and 
1034         non-constant. 
1035  
1036     lmbda : scalar, optional 
1037         If `lmbda` is None (default), find the value of `lmbda` that maximizes 
1038         the log-likelihood function and return it as the second output 
1039         argument. 
1040  
1041         If `lmbda` is not None, do the transformation for that value. 
1042  
1043     alpha : float, optional 
1044         If `lmbda` is None and `alpha` is not None (default), return the 
1045         ``100 * (1-alpha)%`` confidence  interval for `lmbda` as the third 
1046         output argument. Must be between 0.0 and 1.0. 
1047  
1048         If `lmbda` is not None, `alpha` is ignored. 
1049     optimizer : callable, optional 
1050         If `lmbda` is None, `optimizer` is the scalar optimizer used to find 
1051         the value of `lmbda` that minimizes the negative log-likelihood 
1052         function. `optimizer` is a callable that accepts one argument: 
1053  
1054         fun : callable 
1055             The objective function, which evaluates the negative 
1056             log-likelihood function at a provided value of `lmbda` 
1057  
1058         and returns an object, such as an instance of 
1059         `scipy.optimize.OptimizeResult`, which holds the optimal value of 
1060         `lmbda` in an attribute `x`. 
1061  
1062         See the example in `boxcox_normmax` or the documentation of 
1063         `scipy.optimize.minimize_scalar` for more information. 
1064  
1065         If `lmbda` is not None, `optimizer` is ignored. 
1066  
1067     Returns 
1068     ------- 
1069     boxcox : ndarray 
1070         Box-Cox power transformed array. 
1071     maxlog : float, optional 
1072         If the `lmbda` parameter is None, the second returned argument is 
1073         the `lmbda` that maximizes the log-likelihood function. 
1074     (min_ci, max_ci) : tuple of float, optional 
1075         If `lmbda` parameter is None and `alpha` is not None, this returned 
1076         tuple of floats represents the minimum and maximum confidence limits 
1077         given `alpha`. 
1078  
1079     See Also 
1080     -------- 
1081     probplot, boxcox_normplot, boxcox_normmax, boxcox_llf 
1082  
1083     Notes 
1084     ----- 
1085     The Box-Cox transform is given by:: 
1086  
1087         y = (x**lmbda - 1) / lmbda,  for lmbda != 0 
1088             log(x),                  for lmbda = 0 
1089  
1090     `boxcox` requires the input data to be positive.  Sometimes a Box-Cox 
1091     transformation provides a shift parameter to achieve this; `boxcox` does 
1092     not.  Such a shift parameter is equivalent to adding a positive constant to 
1093     `x` before calling `boxcox`. 
1094  
1095     The confidence limits returned when `alpha` is provided give the interval 
1096     where: 
1097  
1098     .. math:: 
1099  
1100         llf(\hat{\lambda}) - llf(\lambda) < \frac{1}{2}\chi^2(1 - \alpha, 1), 
1101  
1102     with ``llf`` the log-likelihood function and :math:`\chi^2` the chi-squared 
1103     function. 
1104  
1105     References 
1106     ---------- 
1107     G.E.P. Box and D.R. Cox, "An Analysis of Transformations", Journal of the 
1108     Royal Statistical Society B, 26, 211-252 (1964). 
1109  
1110     Examples 
1111     -------- 
1112     >>> from scipy import stats 
1113     >>> import matplotlib.pyplot as plt 
1114  
1115     We generate some random variates from a non-normal distribution and make a 
1116     probability plot for it, to show it is non-normal in the tails: 
1117  
1118     >>> fig = plt.figure() 
1119     >>> ax1 = fig.add_subplot(211) 
1120     >>> x = stats.loggamma.rvs(5, size=500) + 5 
1121     >>> prob = stats.probplot(x, dist=stats.norm, plot=ax1) 
1122     >>> ax1.set_xlabel('') 
1123     >>> ax1.set_title('Probplot against normal distribution') 
1124  
1125     We now use `boxcox` to transform the data so it's closest to normal: 
1126  
1127     >>> ax2 = fig.add_subplot(212) 
1128     >>> xt, _ = stats.boxcox(x) 
1129     >>> prob = stats.probplot(xt, dist=stats.norm, plot=ax2) 
1130     >>> ax2.set_title('Probplot after Box-Cox transformation') 
1131  
1132     >>> plt.show() 
1133  
1134     """
1135     x = np.asarray(x)
1136 
1137     if lmbda is not None:  # single transformation
1138         return special.boxcox(x, lmbda)
1139 
1140     if x.ndim != 1:
1141         raise ValueError("Data must be 1-dimensional.")
1142 
1143     if x.size == 0:
1144         return x
1145 
1146     if np.all(x == x[0]):
1147         raise ValueError("Data must not be constant.")
1148 
1149     if np.any(x <= 0):
1150         raise ValueError("Data must be positive.")
1151 
1152     # If lmbda=None, find the lmbda that maximizes the log-likelihood function.
1153     lmax = boxcox_normmax(x, method='mle', optimizer=optimizer)
1154     y = boxcox(x, lmax)
1155 
1156     if alpha is None:
1157         return y, lmax
1158     else:
1159         # Find confidence interval
1160         interval = _boxcox_conf_interval(x, lmax, alpha)
1161         return y, lmax, interval

1179     x, brack=None, method='pearsonr', optimizer=None, *, ymax=_BigFloat_singleton
1180 ):
1181     """Compute optimal Box-Cox transform parameter for input data. 
1182  
1183     Parameters 
1184     ---------- 
1185     x : array_like 
1186         Input array. All entries must be positive, finite, real numbers. 
1187     brack : 2-tuple, optional, default (-2.0, 2.0) 
1188          The starting interval for a downhill bracket search for the default 
1189          `optimize.brent` solver. Note that this is in most cases not 
1190          critical; the final result is allowed to be outside this bracket. 
1191          If `optimizer` is passed, `brack` must be None. 
1192     method : str, optional 
1193         The method to determine the optimal transform parameter (`boxcox` 
1194         ``lmbda`` parameter). Options are: 
1195  
1196         'pearsonr'  (default) 
1197             Maximizes the Pearson correlation coefficient between 
1198             ``y = boxcox(x)`` and the expected values for ``y`` if `x` would be 
1199             normally-distributed. 
1200  
1201         'mle' 
1202             Maximizes the log-likelihood `boxcox_llf`.  This is the method used 
1203             in `boxcox`. 
1204  
1205         'all' 
1206             Use all optimization methods available, and return all results. 
1207             Useful to compare different methods. 
1208     optimizer : callable, optional 
1209         `optimizer` is a callable that accepts one argument: 
1210  
1211         fun : callable 
1212             The objective function to be minimized. `fun` accepts one argument, 
1213             the Box-Cox transform parameter `lmbda`, and returns the value of 
1214             the function (e.g., the negative log-likelihood) at the provided 
1215             argument. The job of `optimizer` is to find the value of `lmbda` 
1216             that *minimizes* `fun`. 
1217  
1218         and returns an object, such as an instance of 
1219         `scipy.optimize.OptimizeResult`, which holds the optimal value of 
1220         `lmbda` in an attribute `x`. 
1221  
1222         See the example below or the documentation of 
1223         `scipy.optimize.minimize_scalar` for more information. 
1224     ymax : float, optional 
1225         The unconstrained optimal transform parameter may cause Box-Cox 
1226         transformed data to have extreme magnitude or even overflow. 
1227         This parameter constrains MLE optimization such that the magnitude 
1228         of the transformed `x` does not exceed `ymax`. The default is 
1229         the maximum value of the input dtype. If set to infinity, 
1230         `boxcox_normmax` returns the unconstrained optimal lambda. 
1231         Ignored when ``method='pearsonr'``. 
1232  
1233     Returns 
1234     ------- 
1235     maxlog : float or ndarray 
1236         The optimal transform parameter found.  An array instead of a scalar 
1237         for ``method='all'``. 
1238  
1239     See Also 
1240     -------- 
1241     boxcox, boxcox_llf, boxcox_normplot, scipy.optimize.minimize_scalar 
1242  
1243     Examples 
1244     -------- 
1245     >>> import numpy as np 
1246     >>> from scipy import stats 
1247     >>> import matplotlib.pyplot as plt 
1248  
1249     We can generate some data and determine the optimal ``lmbda`` in various 
1250     ways: 
1251  
1252     >>> rng = np.random.default_rng() 
1253     >>> x = stats.loggamma.rvs(5, size=30, random_state=rng) + 5 
1254     >>> y, lmax_mle = stats.boxcox(x) 
1255     >>> lmax_pearsonr = stats.boxcox_normmax(x) 
1256  
1257     >>> lmax_mle 
1258     2.217563431465757 
1259     >>> lmax_pearsonr 
1260     2.238318660200961 
1261     >>> stats.boxcox_normmax(x, method='all') 
1262     array([2.23831866, 2.21756343]) 
1263  
1264     >>> fig = plt.figure() 
1265     >>> ax = fig.add_subplot(111) 
1266     >>> prob = stats.boxcox_normplot(x, -10, 10, plot=ax) 
1267     >>> ax.axvline(lmax_mle, color='r') 
1268     >>> ax.axvline(lmax_pearsonr, color='g', ls='--') 
1269  
1270     >>> plt.show() 
1271  
1272     Alternatively, we can define our own `optimizer` function. Suppose we 
1273     are only interested in values of `lmbda` on the interval [6, 7], we 
1274     want to use `scipy.optimize.minimize_scalar` with ``method='bounded'``, 
1275     and we want to use tighter tolerances when optimizing the log-likelihood 
1276     function. To do this, we define a function that accepts positional argument 
1277     `fun` and uses `scipy.optimize.minimize_scalar` to minimize `fun` subject 
1278     to the provided bounds and tolerances: 
1279  
1280     >>> from scipy import optimize 
1281     >>> options = {'xatol': 1e-12}  # absolute tolerance on `x` 
1282     >>> def optimizer(fun): 
1283     ...     return optimize.minimize_scalar(fun, bounds=(6, 7), 
1284     ...                                     method="bounded", options=options) 
1285     >>> stats.boxcox_normmax(x, optimizer=optimizer) 
1286     6.000000000 
1287     """
1288     x = np.asarray(x)
1289 
1290     if not np.all(np.isfinite(x) & (x >= 0)):
1291         message = ("The `x` argument of `boxcox_normmax` must contain "
1292                    "only positive, finite, real numbers.")
1293         raise ValueError(message)
1294 
1295     end_msg = "exceed specified `ymax`."
1296     if ymax is _BigFloat_singleton:
1297         dtype = x.dtype if np.issubdtype(x.dtype, np.floating) else np.float64
1298         # 10000 is a safety factor because `special.boxcox` overflows prematurely.
1299         ymax = np.finfo(dtype).max / 10000
1300         end_msg = f"overflow in {dtype}."
1301     elif ymax <= 0:
1302         raise ValueError("`ymax` must be strictly positive")
1303 
1304     # If optimizer is not given, define default 'brent' optimizer.
1305     if optimizer is None:
1306 
1307         # Set default value for `brack`.
1308         if brack is None:
1309             brack = (-2.0, 2.0)
1310 
1311         def _optimizer(func, args):
1312             return optimize.brent(func, args=args, brack=brack)
1313 
1314     # Otherwise check optimizer.
1315     else:
1316         if not callable(optimizer):
1317             raise ValueError("`optimizer` must be a callable")
1318 
1319         if brack is not None:
1320             raise ValueError("`brack` must be None if `optimizer` is given")
1321 
1322         # `optimizer` is expected to return a `OptimizeResult` object, we here
1323         # get the solution to the optimization problem.
1324         def _optimizer(func, args):
1325             def func_wrapped(x):
1326                 return func(x, *args)
1327             return getattr(optimizer(func_wrapped), 'x', None)
1328 
1329     def _pearsonr(x):
1330         osm_uniform = _calc_uniform_order_statistic_medians(len(x))
1331         xvals = distributions.norm.ppf(osm_uniform)
1332 
1333         def _eval_pearsonr(lmbda, xvals, samps):
1334             # This function computes the x-axis values of the probability plot
1335             # and computes a linear regression (including the correlation) and
1336             # returns ``1 - r`` so that a minimization function maximizes the
1337             # correlation.
1338             y = boxcox(samps, lmbda)
1339             yvals = np.sort(y)
1340             r, prob = _stats_py.pearsonr(xvals, yvals)
1341             return 1 - r
1342 
1343         return _optimizer(_eval_pearsonr, args=(xvals, x))
1344 
1345     def _mle(x):
1346         def _eval_mle(lmb, data):
1347             # function to minimize
1348             return -boxcox_llf(lmb, data)
1349 
1350         return _optimizer(_eval_mle, args=(x,))
1351 
1352     def _all(x):
1353         maxlog = np.empty(2, dtype=float)
1354         maxlog[0] = _pearsonr(x)
1355         maxlog[1] = _mle(x)
1356         return maxlog
1357 
1358     methods = {'pearsonr': _pearsonr,
1359                'mle': _mle,
1360                'all': _all}
1361     if method not in methods.keys():
1362         raise ValueError(f"Method {method} not recognized.")
1363 
1364     optimfunc = methods[method]
1365 
1366     res = optimfunc(x)
1367 
1368     if res is None:
1369         message = ("The `optimizer` argument of `boxcox_normmax` must return "
1370                    "an object containing the optimal `lmbda` in attribute `x`.")
1371         raise ValueError(message)
1372     elif not np.isinf(ymax):  # adjust the final lambda
1373         # x > 1, boxcox(x) > 0; x < 1, boxcox(x) < 0
1374         xmax, xmin = np.max(x), np.min(x)
1375         if xmin >= 1:
1376             x_treme = xmax
1377         elif xmax <= 1:
1378             x_treme = xmin
1379         else:  # xmin < 1 < xmax
1380             indicator = special.boxcox(xmax, res) > abs(special.boxcox(xmin, res))
1381             if isinstance(res, np.ndarray):
1382                 indicator = indicator[1]  # select corresponds with 'mle'
1383             x_treme = xmax if indicator else xmin
1384 
1385         mask = abs(special.boxcox(x_treme, res)) > ymax
1386         if np.any(mask):
1387             message = (
1388                 f"The optimal lambda is {res}, but the returned lambda is the "
1389                 f"constrained optimum to ensure that the maximum or the minimum "
1390                 f"of the transformed data does not " + end_msg
1391             )
1392             warnings.warn(message, stacklevel=2)
1393 
1394             # Return the constrained lambda to ensure the transformation
1395             # does not cause overflow or exceed specified `ymax`
1396             constrained_res = _boxcox_inv_lmbda(x_treme, ymax * np.sign(x_treme - 1))
1397 
1398             if isinstance(res, np.ndarray):
1399                 res[mask] = constrained_res
1400             else:
1401                 res = constrained_res
1402     return res

1515 def yeojohnson(x, lmbda=None):
1516     r"""Return a dataset transformed by a Yeo-Johnson power transformation. 
1517  
1518     Parameters 
1519     ---------- 
1520     x : ndarray 
1521         Input array.  Should be 1-dimensional. 
1522     lmbda : float, optional 
1523         If ``lmbda`` is ``None``, find the lambda that maximizes the 
1524         log-likelihood function and return it as the second output argument. 
1525         Otherwise the transformation is done for the given value. 
1526  
1527     Returns 
1528     ------- 
1529     yeojohnson: ndarray 
1530         Yeo-Johnson power transformed array. 
1531     maxlog : float, optional 
1532         If the `lmbda` parameter is None, the second returned argument is 
1533         the lambda that maximizes the log-likelihood function. 
1534  
1535     See Also 
1536     -------- 
1537     probplot, yeojohnson_normplot, yeojohnson_normmax, yeojohnson_llf, boxcox 
1538  
1539     Notes 
1540     ----- 
1541     The Yeo-Johnson transform is given by:: 
1542  
1543         y = ((x + 1)**lmbda - 1) / lmbda,                for x >= 0, lmbda != 0 
1544             log(x + 1),                                  for x >= 0, lmbda = 0 
1545             -((-x + 1)**(2 - lmbda) - 1) / (2 - lmbda),  for x < 0, lmbda != 2 
1546             -log(-x + 1),                                for x < 0, lmbda = 2 
1547  
1548     Unlike `boxcox`, `yeojohnson` does not require the input data to be 
1549     positive. 
1550  
1551     .. versionadded:: 1.2.0 
1552  
1553  
1554     References 
1555     ---------- 
1556     I. Yeo and R.A. Johnson, "A New Family of Power Transformations to 
1557     Improve Normality or Symmetry", Biometrika 87.4 (2000): 
1558  
1559  
1560     Examples 
1561     -------- 
1562     >>> from scipy import stats 
1563     >>> import matplotlib.pyplot as plt 
1564  
1565     We generate some random variates from a non-normal distribution and make a 
1566     probability plot for it, to show it is non-normal in the tails: 
1567  
1568     >>> fig = plt.figure() 
1569     >>> ax1 = fig.add_subplot(211) 
1570     >>> x = stats.loggamma.rvs(5, size=500) + 5 
1571     >>> prob = stats.probplot(x, dist=stats.norm, plot=ax1) 
1572     >>> ax1.set_xlabel('') 
1573     >>> ax1.set_title('Probplot against normal distribution') 
1574  
1575     We now use `yeojohnson` to transform the data so it's closest to normal: 
1576  
1577     >>> ax2 = fig.add_subplot(212) 
1578     >>> xt, lmbda = stats.yeojohnson(x) 
1579     >>> prob = stats.probplot(xt, dist=stats.norm, plot=ax2) 
1580     >>> ax2.set_title('Probplot after Yeo-Johnson transformation') 
1581  
1582     >>> plt.show() 
1583  
1584     """
1585     x = np.asarray(x)
1586     if x.size == 0:
1587         return x
1588 
1589     if np.issubdtype(x.dtype, np.complexfloating):
1590         raise ValueError('Yeo-Johnson transformation is not defined for '
1591                          'complex numbers.')
1592 
1593     if np.issubdtype(x.dtype, np.integer):
1594         x = x.astype(np.float64, copy=False)
1595 
1596     if lmbda is not None:
1597         return _yeojohnson_transform(x, lmbda)
1598 
1599     # if lmbda=None, find the lmbda that maximizes the log-likelihood function.
1600     lmax = yeojohnson_normmax(x)
1601     y = _yeojohnson_transform(x, lmax)
1602 
1603     return y, lmax

1606 def _yeojohnson_transform(x, lmbda):
1607     """Returns `x` transformed by the Yeo-Johnson power transform with given 
1608     parameter `lmbda`. 
1609     """
1610     dtype = x.dtype if np.issubdtype(x.dtype, np.floating) else np.float64
1611     out = np.zeros_like(x, dtype=dtype)
1612     pos = x >= 0  # binary mask
1613 
1614     # when x >= 0
1615     if abs(lmbda) < np.spacing(1.):
1616         out[pos] = np.log1p(x[pos])
1617     else:  # lmbda != 0
1618         # more stable version of: ((x + 1) ** lmbda - 1) / lmbda
1619         out[pos] = np.expm1(lmbda * np.log1p(x[pos])) / lmbda
1620 
1621     # when x < 0
1622     if abs(lmbda - 2) > np.spacing(1.):
1623         out[~pos] = -np.expm1((2 - lmbda) * np.log1p(-x[~pos])) / (2 - lmbda)
1624     else:  # lmbda == 2
1625         out[~pos] = -np.log1p(-x[~pos])
1626 
1627     return out

1630 def yeojohnson_llf(lmb, data):
1631     r"""The yeojohnson log-likelihood function. 
1632  
1633     Parameters 
1634     ---------- 
1635     lmb : scalar 
1636         Parameter for Yeo-Johnson transformation. See `yeojohnson` for 
1637         details. 
1638     data : array_like 
1639         Data to calculate Yeo-Johnson log-likelihood for. If `data` is 
1640         multi-dimensional, the log-likelihood is calculated along the first 
1641         axis. 
1642  
1643     Returns 
1644     ------- 
1645     llf : float 
1646         Yeo-Johnson log-likelihood of `data` given `lmb`. 
1647  
1648     See Also 
1649     -------- 
1650     yeojohnson, probplot, yeojohnson_normplot, yeojohnson_normmax 
1651  
1652     Notes 
1653     ----- 
1654     The Yeo-Johnson log-likelihood function is defined here as 
1655  
1656     .. math:: 
1657  
1658         llf = -N/2 \log(\hat{\sigma}^2) + (\lambda - 1) 
1659               \sum_i \text{ sign }(x_i)\log(|x_i| + 1) 
1660  
1661     where :math:`\hat{\sigma}^2` is estimated variance of the Yeo-Johnson 
1662     transformed input data ``x``. 
1663  
1664     .. versionadded:: 1.2.0 
1665  
1666     Examples 
1667     -------- 
1668     >>> import numpy as np 
1669     >>> from scipy import stats 
1670     >>> import matplotlib.pyplot as plt 
1671     >>> from mpl_toolkits.axes_grid1.inset_locator import inset_axes 
1672  
1673     Generate some random variates and calculate Yeo-Johnson log-likelihood 
1674     values for them for a range of ``lmbda`` values: 
1675  
1676     >>> x = stats.loggamma.rvs(5, loc=10, size=1000) 
1677     >>> lmbdas = np.linspace(-2, 10) 
1678     >>> llf = np.zeros(lmbdas.shape, dtype=float) 
1679     >>> for ii, lmbda in enumerate(lmbdas): 
1680     ...     llf[ii] = stats.yeojohnson_llf(lmbda, x) 
1681  
1682     Also find the optimal lmbda value with `yeojohnson`: 
1683  
1684     >>> x_most_normal, lmbda_optimal = stats.yeojohnson(x) 
1685  
1686     Plot the log-likelihood as function of lmbda.  Add the optimal lmbda as a 
1687     horizontal line to check that that's really the optimum: 
1688  
1689     >>> fig = plt.figure() 
1690     >>> ax = fig.add_subplot(111) 
1691     >>> ax.plot(lmbdas, llf, 'b.-') 
1692     >>> ax.axhline(stats.yeojohnson_llf(lmbda_optimal, x), color='r') 
1693     >>> ax.set_xlabel('lmbda parameter') 
1694     >>> ax.set_ylabel('Yeo-Johnson log-likelihood') 
1695  
1696     Now add some probability plots to show that where the log-likelihood is 
1697     maximized the data transformed with `yeojohnson` looks closest to normal: 
1698  
1699     >>> locs = [3, 10, 4]  # 'lower left', 'center', 'lower right' 
1700     >>> for lmbda, loc in zip([-1, lmbda_optimal, 9], locs): 
1701     ...     xt = stats.yeojohnson(x, lmbda=lmbda) 
1702     ...     (osm, osr), (slope, intercept, r_sq) = stats.probplot(xt) 
1703     ...     ax_inset = inset_axes(ax, width="20%", height="20%", loc=loc) 
1704     ...     ax_inset.plot(osm, osr, 'c.', osm, slope*osm + intercept, 'k-') 
1705     ...     ax_inset.set_xticklabels([]) 
1706     ...     ax_inset.set_yticklabels([]) 
1707     ...     ax_inset.set_title(r'$\lambda=%1.2f$' % lmbda) 
1708  
1709     >>> plt.show() 
1710  
1711     """
1712     data = np.asarray(data)
1713     n_samples = data.shape[0]
1714 
1715     if n_samples == 0:
1716         return np.nan
1717 
1718     trans = _yeojohnson_transform(data, lmb)
1719     trans_var = trans.var(axis=0)
1720     loglike = np.empty_like(trans_var)
1721 
1722     # Avoid RuntimeWarning raised by np.log when the variance is too low
1723     tiny_variance = trans_var < np.finfo(trans_var.dtype).tiny
1724     loglike[tiny_variance] = np.inf
1725 
1726     loglike[~tiny_variance] = (
1727         -n_samples / 2 * np.log(trans_var[~tiny_variance]))
1728     loglike[~tiny_variance] += (
1729         (lmb - 1) * (np.sign(data) * np.log1p(np.abs(data))).sum(axis=0))
1730     return loglike
1731

1733 def yeojohnson_normmax(x, brack=None):
1734     """Compute optimal Yeo-Johnson transform parameter. 
1735  
1736     Compute optimal Yeo-Johnson transform parameter for input data, using 
1737     maximum likelihood estimation. 
1738  
1739     Parameters 
1740     ---------- 
1741     x : array_like 
1742         Input array. 
1743     brack : 2-tuple, optional 
1744         The starting interval for a downhill bracket search with 
1745         `optimize.brent`. Note that this is in most cases not critical; the 
1746         final result is allowed to be outside this bracket. If None, 
1747         `optimize.fminbound` is used with bounds that avoid overflow. 
1748  
1749     Returns 
1750     ------- 
1751     maxlog : float 
1752         The optimal transform parameter found. 
1753  
1754     See Also 
1755     -------- 
1756     yeojohnson, yeojohnson_llf, yeojohnson_normplot 
1757  
1758     Notes 
1759     ----- 
1760     .. versionadded:: 1.2.0 
1761  
1762     Examples 
1763     -------- 
1764     >>> import numpy as np 
1765     >>> from scipy import stats 
1766     >>> import matplotlib.pyplot as plt 
1767  
1768     Generate some data and determine optimal ``lmbda`` 
1769  
1770     >>> rng = np.random.default_rng() 
1771     >>> x = stats.loggamma.rvs(5, size=30, random_state=rng) + 5 
1772     >>> lmax = stats.yeojohnson_normmax(x) 
1773  
1774     >>> fig = plt.figure() 
1775     >>> ax = fig.add_subplot(111) 
1776     >>> prob = stats.yeojohnson_normplot(x, -10, 10, plot=ax) 
1777     >>> ax.axvline(lmax, color='r') 
1778  
1779     >>> plt.show() 
1780  
1781     """
1782     def _neg_llf(lmbda, data):
1783         llf = yeojohnson_llf(lmbda, data)
1784         # reject likelihoods that are inf which are likely due to small
1785         # variance in the transformed space
1786         llf[np.isinf(llf)] = -np.inf
1787         return -llf
1788 
1789     with np.errstate(invalid='ignore'):
1790         if not np.all(np.isfinite(x)):
1791             raise ValueError('Yeo-Johnson input must be finite.')
1792         if np.all(x == 0):
1793             return 1.0
1794         if brack is not None:
1795             return optimize.brent(_neg_llf, brack=brack, args=(x,))
1796         x = np.asarray(x)
1797         dtype = x.dtype if np.issubdtype(x.dtype, np.floating) else np.float64
1798         # Allow values up to 20 times the maximum observed value to be safely
1799         # transformed without over- or underflow.
1800         log1p_max_x = np.log1p(20 * np.max(np.abs(x)))
1801         # Use half of floating point's exponent range to allow safe computation
1802         # of the variance of the transformed data.
1803         log_eps = np.log(np.finfo(dtype).eps)
1804         log_tiny_float = (np.log(np.finfo(dtype).tiny) - log_eps) / 2
1805         log_max_float = (np.log(np.finfo(dtype).max) + log_eps) / 2
1806         # Compute the bounds by approximating the inverse of the Yeo-Johnson
1807         # transform on the smallest and largest floating point exponents, given
1808         # the largest data we expect to observe. See [1] for further details.
1809         # [1] https://github.com/scipy/scipy/pull/18852#issuecomment-1630286174
1810         lb = log_tiny_float / log1p_max_x
1811         ub = log_max_float / log1p_max_x
1812         # Convert the bounds if all or some of the data is negative.
1813         if np.all(x < 0):
1814             lb, ub = 2 - ub, 2 - lb
1815         elif np.any(x < 0):
1816             lb, ub = max(2 - ub, lb), min(2 - lb, ub)
1817         # Match `optimize.brent`'s tolerance.
1818         tol_brent = 1.48e-08
1819         return optimize.fminbound(_neg_llf, lb, ub, args=(x,), xtol=tol_brent)

2019 def _weibull_fit_check(params, x):
2020     # Refine the fit returned by `weibull_min.fit` to ensure that the first
2021     # order necessary conditions are satisfied. If not, raise an error.
2022     # Here, use `m` for the shape parameter to be consistent with [7]
2023     # and avoid confusion with `c` as defined in [7].
2024     n = len(x)
2025     m, u, s = params
2026 
2027     def dnllf_dm(m, u):
2028         # Partial w.r.t. shape w/ optimal scale. See [7] Equation 5.
2029         xu = x-u
2030         return (1/m - (xu**m*np.log(xu)).sum()/(xu**m).sum()
2031                 + np.log(xu).sum()/n)
2032 
2033     def dnllf_du(m, u):
2034         # Partial w.r.t. loc w/ optimal scale. See [7] Equation 6.
2035         xu = x-u
2036         return (m-1)/m*(xu**-1).sum() - n*(xu**(m-1)).sum()/(xu**m).sum()
2037 
2038     def get_scale(m, u):
2039         # Partial w.r.t. scale solved in terms of shape and location.
2040         # See [7] Equation 7.
2041         return ((x-u)**m/n).sum()**(1/m)
2042 
2043     def dnllf(params):
2044         # Partial derivatives of the NLLF w.r.t. parameters, i.e.
2045         # first order necessary conditions for MLE fit.
2046         return [dnllf_dm(*params), dnllf_du(*params)]

2049                   "for the three-parameter Weibull distribution. Consider "
2050                   "performing a custom goodness-of-fit test using "
2051                   "`scipy.stats.monte_carlo_test`.")

2053     if np.allclose(u, np.min(x)) or m < 1:
2054         # The critical values provided by [7] don't seem to control the
2055         # Type I error rate in this case. Error out.
2056         message = ("Maximum likelihood estimation has converged to "
2057                    "a solution in which the location is equal to the minimum "
2058                    "of the data, the shape parameter is less than 2, or both. "
2059                    "The table of critical values in [7] does not "
2060                    "include this case. " + suggestion)

2063     try:

2299     samples : sequence of 1-D array_like 
2300         Array of sample arrays. 
2301     Z : array_like 
2302         Sorted array of all observations. 
2303     Zstar : array_like

2306         Number of samples. 
2307     n : array_like 
2308         Number of observations in each sample. 
2309     N : int 
2310         Total number of observations. 
2311  
2312     Returns 
2313     ------- 
2314     A2aKN : float 
2315         The A2aKN statistics of Scholz and Stephens 1987. 
2316  
2317     """
2318     A2akN = 0.
2319     Z_ssorted_left = Z.searchsorted(Zstar, 'left')
2320     if N == Zstar.size:
2321         lj = 1.
2322     else:
2323         lj = Z.searchsorted(Zstar, 'right') - Z_ssorted_left
2324     Bj = Z_ssorted_left + lj / 2.
2325     for i in arange(0, k):
2326         s = np.sort(samples[i])
2327         s_ssorted_right = s.searchsorted(Zstar, side='right')
2328         Mij = s_ssorted_right.astype(float)
2329         fij = s_ssorted_right - s.searchsorted(Zstar, 'left')
2330         Mij -= fij / 2.
2331         inner = lj / float(N) * (N*Mij - Bj*n[i])**2 / (Bj*(N - Bj) - N*lj/4.)
2332         A2akN += inner.sum() / n[i]
2333     A2akN *= (N - 1.) / N
2334     return A2akN

2379 def anderson_ksamp(samples, midrank=True, *, method=None):
2380     """The Anderson-Darling test for k-samples. 
2381  
2382     The k-sample Anderson-Darling test is a modification of the 
2383     one-sample Anderson-Darling test. It tests the null hypothesis 
2384     that k-samples are drawn from the same population without having 
2385     to specify the distribution function of that population. The 
2386     critical values depend on the number of samples. 
2387  
2388     Parameters 
2389     ---------- 
2390     samples : sequence of 1-D array_like 
2391         Array of sample data in arrays. 
2392     midrank : bool, optional 
2393         Type of Anderson-Darling test which is computed. Default 
2394         (True) is the midrank test applicable to continuous and 
2395         discrete populations. If False, the right side empirical 
2396         distribution is used. 
2397     method : PermutationMethod, optional 
2398         Defines the method used to compute the p-value. If `method` is an 
2399         instance of `PermutationMethod`, the p-value is computed using 
2400         `scipy.stats.permutation_test` with the provided configuration options 
2401         and other appropriate settings. Otherwise, the p-value is interpolated 
2402         from tabulated values. 
2403  
2404     Returns 
2405     ------- 
2406     res : Anderson_ksampResult 
2407         An object containing attributes: 
2408  
2409         statistic : float 
2410             Normalized k-sample Anderson-Darling test statistic. 
2411         critical_values : array 
2412             The critical values for significance levels 25%, 10%, 5%, 2.5%, 1%, 
2413             0.5%, 0.1%. 
2414         pvalue : float 
2415             The approximate p-value of the test. If `method` is not 
2416             provided, the value is floored / capped at 0.1% / 25%. 
2417  
2418     Raises 
2419     ------ 
2420     ValueError 
2421         If fewer than 2 samples are provided, a sample is empty, or no 
2422         distinct observations are in the samples. 
2423  
2424     See Also 
2425     -------- 
2426     ks_2samp : 2 sample Kolmogorov-Smirnov test 
2427     anderson : 1 sample Anderson-Darling test 
2428  
2429     Notes 
2430     ----- 
2431     [1]_ defines three versions of the k-sample Anderson-Darling test: 
2432     one for continuous distributions and two for discrete 
2433     distributions, in which ties between samples may occur. The 
2434     default of this routine is to compute the version based on the 
2435     midrank empirical distribution function. This test is applicable 
2436     to continuous and discrete data. If midrank is set to False, the 
2437     right side empirical distribution is used for a test for discrete 
2438     data. According to [1]_, the two discrete test statistics differ 
2439     only slightly if a few collisions due to round-off errors occur in 
2440     the test not adjusted for ties between samples. 
2441  
2442     The critical values corresponding to the significance levels from 0.01 
2443     to 0.25 are taken from [1]_. p-values are floored / capped 
2444     at 0.1% / 25%. Since the range of critical values might be extended in 
2445     future releases, it is recommended not to test ``p == 0.25``, but rather 
2446     ``p >= 0.25`` (analogously for the lower bound). 
2447  
2448     .. versionadded:: 0.14.0 
2449  
2450     References 
2451     ---------- 
2452     .. [1] Scholz, F. W and Stephens, M. A. (1987), K-Sample 
2453            Anderson-Darling Tests, Journal of the American Statistical 
2454            Association, Vol. 82, pp. 918-924. 
2455  
2456     Examples 
2457     -------- 
2458     >>> import numpy as np 
2459     >>> from scipy import stats 
2460     >>> rng = np.random.default_rng() 
2461     >>> res = stats.anderson_ksamp([rng.normal(size=50), 
2462     ... rng.normal(loc=0.5, size=30)]) 
2463     >>> res.statistic, res.pvalue 
2464     (1.974403288713695, 0.04991293614572478) 
2465     >>> res.critical_values 
2466     array([0.325, 1.226, 1.961, 2.718, 3.752, 4.592, 6.546]) 
2467  
2468     The null hypothesis that the two random samples come from the same 
2469     distribution can be rejected at the 5% level because the returned 
2470     test value is greater than the critical value for 5% (1.961) but 
2471     not at the 2.5% level. The interpolation gives an approximate 
2472     p-value of 4.99%. 
2473  
2474     >>> samples = [rng.normal(size=50), rng.normal(size=30), 
2475     ...            rng.normal(size=20)] 
2476     >>> res = stats.anderson_ksamp(samples) 
2477     >>> res.statistic, res.pvalue 
2478     (-0.29103725200789504, 0.25) 
2479     >>> res.critical_values 
2480     array([ 0.44925884,  1.3052767 ,  1.9434184 ,  2.57696569,  3.41634856, 
2481       4.07210043, 5.56419101]) 
2482  
2483     The null hypothesis cannot be rejected for three samples from an 
2484     identical distribution. The reported p-value (25%) has been capped and 
2485     may not be very accurate (since it corresponds to the value 0.449 
2486     whereas the statistic is -0.291). 
2487  
2488     In such cases where the p-value is capped or when sample sizes are 
2489     small, a permutation test may be more accurate. 
2490  
2491     >>> method = stats.PermutationMethod(n_resamples=9999, random_state=rng) 
2492     >>> res = stats.anderson_ksamp(samples, method=method) 
2493     >>> res.pvalue 
2494     0.5254 
2495  
2496     """
2497     k = len(samples)
2498     if (k < 2):
2499         raise ValueError("anderson_ksamp needs at least two samples")
2500 
2501     samples = list(map(np.asarray, samples))
2502     Z = np.sort(np.hstack(samples))
2503     N = Z.size
2504     Zstar = np.unique(Z)
2505     if Zstar.size < 2:
2506         raise ValueError("anderson_ksamp needs more than one distinct "
2507                          "observation")
2508 
2509     n = np.array([sample.size for sample in samples])
2510     if np.any(n == 0):
2511         raise ValueError("anderson_ksamp encountered sample without "
2512                          "observations")
2513 
2514     if midrank:
2515         A2kN_fun = _anderson_ksamp_midrank
2516     else:
2517         A2kN_fun = _anderson_ksamp_right
2518     A2kN = A2kN_fun(samples, Z, Zstar, k, n, N)
2519 
2520     def statistic(*samples):
2521         return A2kN_fun(samples, Z, Zstar, k, n, N)
2522 
2523     if method is not None:
2524         res = stats.permutation_test(samples, statistic, **method._asdict(),
2525                                      alternative='greater')
2526 
2527     H = (1. / n).sum()
2528     hs_cs = (1. / arange(N - 1, 1, -1)).cumsum()
2529     h = hs_cs[-1] + 1
2530     g = (hs_cs / arange(2, N)).sum()
2531 
2532     a = (4*g - 6) * (k - 1) + (10 - 6*g)*H
2533     b = (2*g - 4)*k**2 + 8*h*k + (2*g - 14*h - 4)*H - 8*h + 4*g - 6
2534     c = (6*h + 2*g - 2)*k**2 + (4*h - 4*g + 6)*k + (2*h - 6)*H + 4*h
2535     d = (2*h + 6)*k**2 - 4*h*k
2536     sigmasq = (a*N**3 + b*N**2 + c*N + d) / ((N - 1.) * (N - 2.) * (N - 3.))
2537     m = k - 1
2538     A2 = (A2kN - m) / math.sqrt(sigmasq)
2539 
2540     # The b_i values are the interpolation coefficients from Table 2
2541     # of Scholz and Stephens 1987
2542     b0 = np.array([0.675, 1.281, 1.645, 1.96, 2.326, 2.573, 3.085])
2543     b1 = np.array([-0.245, 0.25, 0.678, 1.149, 1.822, 2.364, 3.615])
2544     b2 = np.array([-0.105, -0.305, -0.362, -0.391, -0.396, -0.345, -0.154])
2545     critical = b0 + b1 / math.sqrt(m) + b2 / m
2546 
2547     sig = np.array([0.25, 0.1, 0.05, 0.025, 0.01, 0.005, 0.001])
2548 
2549     if A2 < critical.min() and method is None:
2550         p = sig.max()
2551         msg = (f"p-value capped: true value larger than {p}. Consider "
2552                "specifying `method` "
2553                "(e.g. `method=stats.PermutationMethod()`.)")
2554         warnings.warn(msg, stacklevel=2)
2555     elif A2 > critical.max() and method is None:
2556         p = sig.min()
2557         msg = (f"p-value floored: true value smaller than {p}. Consider "
2558                "specifying `method` "
2559                "(e.g. `method=stats.PermutationMethod()`.)")
2560         warnings.warn(msg, stacklevel=2)
2561     elif method is None:
2562         # interpolation of probit of significance level
2563         pf = np.polyfit(critical, log(sig), 2)
2564         p = math.exp(np.polyval(pf, A2))
2565     else:
2566         p = res.pvalue if method is not None else p
2567 
2568     # create result object with alias for backward compatibility
2569     res = Anderson_ksampResult(A2, critical, p)
2570     res.significance_level = p
2571     return res

2592         """When necessary, recalculate exact distribution."""
2593         if n != self.n or m != self.m:
2594             self.n, self.m = n, m
2595             # distribution is NOT symmetric when m + n is odd
2596             # n is len(x), m is len(y), and ratio of scales is defined x/y

2598             self.astart = astart  # minimum value of statistic

2600             # expressed as frequencies/counts/integers to maintain precision.
2601             # Stored as floats to avoid overflow of sums.
2602             self.freqs = a1.astype(np.float64)
2603             self.total = self.freqs.sum()  # could calculate from m and n
2604             # probability mass is self.freqs / self.total;

2608         self._recalc(n, m)
2609         # The convention here is that PMF at k = 12.5 is the same as at k = 12,
2610         # -> use `floor` in case of ties.
2611         ind = np.floor(k - self.astart).astype(int)

2619         ind = np.ceil(k - self.astart).astype(int)

2625         # Null distribution derived without considering ties is
2626         # approximate. Round down to avoid Type I error.
2627         ind = np.floor(k - self.astart).astype(int)

2638 def ansari(x, y, alternative='two-sided'):
2639     """Perform the Ansari-Bradley test for equal scale parameters. 
2640  
2641     The Ansari-Bradley test ([1]_, [2]_) is a non-parametric test 
2642     for the equality of the scale parameter of the distributions 
2643     from which two samples were drawn. The null hypothesis states that 
2644     the ratio of the scale of the distribution underlying `x` to the scale 
2645     of the distribution underlying `y` is 1. 
2646  
2647     Parameters 
2648     ---------- 
2649     x, y : array_like 
2650         Arrays of sample data. 
2651     alternative : {'two-sided', 'less', 'greater'}, optional 
2652         Defines the alternative hypothesis. Default is 'two-sided'. 
2653         The following options are available: 
2654  
2655         * 'two-sided': the ratio of scales is not equal to 1. 
2656         * 'less': the ratio of scales is less than 1. 
2657         * 'greater': the ratio of scales is greater than 1. 
2658  
2659         .. versionadded:: 1.7.0 
2660  
2661     Returns 
2662     ------- 
2663     statistic : float 
2664         The Ansari-Bradley test statistic. 
2665     pvalue : float 
2666         The p-value of the hypothesis test. 
2667  
2668     See Also 
2669     -------- 
2670     fligner : A non-parametric test for the equality of k variances 
2671     mood : A non-parametric test for the equality of two scale parameters 
2672  
2673     Notes 
2674     ----- 
2675     The p-value given is exact when the sample sizes are both less than 
2676     55 and there are no ties, otherwise a normal approximation for the 
2677     p-value is used. 
2678  
2679     References 
2680     ---------- 
2681     .. [1] Ansari, A. R. and Bradley, R. A. (1960) Rank-sum tests for 
2682            dispersions, Annals of Mathematical Statistics, 31, 1174-1189. 
2683     .. [2] Sprent, Peter and N.C. Smeeton.  Applied nonparametric 
2684            statistical methods.  3rd ed. Chapman and Hall/CRC. 2001. 
2685            Section 5.8.2. 
2686     .. [3] Nathaniel E. Helwig "Nonparametric Dispersion and Equality 
2687            Tests" at http://users.stat.umn.edu/~helwig/notes/npde-Notes.pdf 
2688  
2689     Examples 
2690     -------- 
2691     >>> import numpy as np 
2692     >>> from scipy.stats import ansari 
2693     >>> rng = np.random.default_rng() 
2694  
2695     For these examples, we'll create three random data sets.  The first 
2696     two, with sizes 35 and 25, are drawn from a normal distribution with 
2697     mean 0 and standard deviation 2.  The third data set has size 25 and 
2698     is drawn from a normal distribution with standard deviation 1.25. 
2699  
2700     >>> x1 = rng.normal(loc=0, scale=2, size=35) 
2701     >>> x2 = rng.normal(loc=0, scale=2, size=25) 
2702     >>> x3 = rng.normal(loc=0, scale=1.25, size=25) 
2703  
2704     First we apply `ansari` to `x1` and `x2`.  These samples are drawn 
2705     from the same distribution, so we expect the Ansari-Bradley test 
2706     should not lead us to conclude that the scales of the distributions 
2707     are different. 
2708  
2709     >>> ansari(x1, x2) 
2710     AnsariResult(statistic=541.0, pvalue=0.9762532927399098) 
2711  
2712     With a p-value close to 1, we cannot conclude that there is a 
2713     significant difference in the scales (as expected). 
2714  
2715     Now apply the test to `x1` and `x3`: 
2716  
2717     >>> ansari(x1, x3) 
2718     AnsariResult(statistic=425.0, pvalue=0.0003087020407974518) 
2719  
2720     The probability of observing such an extreme value of the statistic 
2721     under the null hypothesis of equal scales is only 0.03087%. We take this 
2722     as evidence against the null hypothesis in favor of the alternative: 
2723     the scales of the distributions from which the samples were drawn 
2724     are not equal. 
2725  
2726     We can use the `alternative` parameter to perform a one-tailed test. 
2727     In the above example, the scale of `x1` is greater than `x3` and so 
2728     the ratio of scales of `x1` and `x3` is greater than 1. This means 
2729     that the p-value when ``alternative='greater'`` should be near 0 and 
2730     hence we should be able to reject the null hypothesis: 
2731  
2732     >>> ansari(x1, x3, alternative='greater') 
2733     AnsariResult(statistic=425.0, pvalue=0.0001543510203987259) 
2734  
2735     As we can see, the p-value is indeed quite low. Use of 
2736     ``alternative='less'`` should thus yield a large p-value: 
2737  
2738     >>> ansari(x1, x3, alternative='less') 
2739     AnsariResult(statistic=425.0, pvalue=0.9998643258449039) 
2740  
2741     """
2742     if alternative not in {'two-sided', 'greater', 'less'}:
2743         raise ValueError("'alternative' must be 'two-sided',"
2744                          " 'greater', or 'less'.")
2745 
2746     if not hasattr(_abw_state, 'a'):
2747         _abw_state.a = _ABW()
2748 
2749     x, y = asarray(x), asarray(y)
2750     n = len(x)
2751     m = len(y)
2752     if m < 1:
2753         raise ValueError("Not enough other observations.")
2754     if n < 1:
2755         raise ValueError("Not enough test observations.")
2756 
2757     N = m + n
2758     xy = r_[x, y]  # combine
2759     rank = _stats_py.rankdata(xy)
2760     symrank = amin(array((rank, N - rank + 1)), 0)
2761     AB = np.sum(symrank[:n], axis=0)
2762     uxy = unique(xy)
2763     repeats = (len(uxy) != len(xy))
2764     exact = ((m < 55) and (n < 55) and not repeats)
2765     if repeats and (m < 55 or n < 55):
2766         warnings.warn("Ties preclude use of exact statistic.", stacklevel=2)
2767     if exact:
2768         if alternative == 'two-sided':
2769             pval = 2.0 * np.minimum(_abw_state.a.cdf(AB, n, m),
2770                                     _abw_state.a.sf(AB, n, m))
2771         elif alternative == 'greater':
2772             # AB statistic is _smaller_ when ratio of scales is larger,
2773             # so this is the opposite of the usual calculation
2774             pval = _abw_state.a.cdf(AB, n, m)
2775         else:
2776             pval = _abw_state.a.sf(AB, n, m)
2777         return AnsariResult(AB, min(1.0, pval))
2778 
2779     # otherwise compute normal approximation
2780     if N % 2:  # N odd
2781         mnAB = n * (N+1.0)**2 / 4.0 / N
2782         varAB = n * m * (N+1.0) * (3+N**2) / (48.0 * N**2)
2783     else:
2784         mnAB = n * (N+2.0) / 4.0
2785         varAB = m * n * (N+2) * (N-2.0) / 48 / (N-1.0)
2786     if repeats:   # adjust variance estimates
2787         # compute np.sum(tj * rj**2,axis=0)
2788         fac = np.sum(symrank**2, axis=0)
2789         if N % 2:  # N odd
2790             varAB = m * n * (16*N*fac - (N+1)**4) / (16.0 * N**2 * (N-1))
2791         else:  # N even
2792             varAB = m * n * (16*fac - N*(N+2)**2) / (16.0 * N * (N-1))
2793 
2794     # Small values of AB indicate larger dispersion for the x sample.
2795     # Large values of AB indicate larger dispersion for the y sample.
2796     # This is opposite to the way we define the ratio of scales. see [1]_.
2797     z = (mnAB - AB) / sqrt(varAB)
2798     pvalue = _get_pvalue(z, _SimpleNormal(), alternative, xp=np)
2799     return AnsariResult(AB[()], pvalue[()])

3221 def _mood_inner_lc(xy, x, diffs, sorted_xy, n, m, N) -> float:
3222     # Obtain the unique values and their frequencies from the pooled samples.
3223     # "a_j, + b_j, = t_j, for j = 1, ... k" where `k` is the number of unique
3224     # classes, and "[t]he number of values associated with the x's and y's in
3225     # the jth class will be denoted by a_j, and b_j respectively."
3226     # (Mielke, 312)
3227     # Reuse previously computed sorted array and `diff` arrays to obtain the
3228     # unique values and counts. Prepend `diffs` with a non-zero to indicate
3229     # that the first element should be marked as not matching what preceded it.
3230     diffs_prep = np.concatenate(([1], diffs))
3231     # Unique elements are where the was a difference between elements in the
3232     # sorted array
3233     uniques = sorted_xy[diffs_prep != 0]
3234     # The count of each element is the bin size for each set of consecutive
3235     # differences where the difference is zero. Replace nonzero differences
3236     # with 1 and then use the cumulative sum to count the indices.
3237     t = np.bincount(np.cumsum(np.asarray(diffs_prep != 0, dtype=int)))[1:]
3238     k = len(uniques)
3239     js = np.arange(1, k + 1, dtype=int)
3240     # the `b` array mentioned in the paper is not used, outside of the
3241     # calculation of `t`, so we do not need to calculate it separately. Here
3242     # we calculate `a`. In plain language, `a[j]` is the number of values in
3243     # `x` that equal `uniques[j]`.
3244     sorted_xyx = np.sort(np.concatenate((xy, x)))
3245     diffs = np.diff(sorted_xyx)
3246     diffs_prep = np.concatenate(([1], diffs))
3247     diff_is_zero = np.asarray(diffs_prep != 0, dtype=int)
3248     xyx_counts = np.bincount(np.cumsum(diff_is_zero))[1:]
3249     a = xyx_counts - t
3250     # "Define .. a_0 = b_0 = t_0 = S_0 = 0" (Mielke 312) so we shift  `a`
3251     # and `t` arrays over 1 to allow a first element of 0 to accommodate this
3252     # indexing.
3253     t = np.concatenate(([0], t))
3254     a = np.concatenate(([0], a))
3255     # S is built from `t`, so it does not need a preceding zero added on.
3256     S = np.cumsum(t)
3257     # define a copy of `S` with a prepending zero for later use to avoid
3258     # the need for indexing.
3259     S_i_m1 = np.concatenate(([0], S[:-1]))
3260 
3261     # Psi, as defined by the 6th unnumbered equation on page 313 (Mielke).
3262     # Note that in the paper there is an error where the denominator `2` is
3263     # squared when it should be the entire equation.
3264     def psi(indicator):
3265         return (indicator - (N + 1)/2)**2
3266 
3267     # define summation range for use in calculation of phi, as seen in sum
3268     # in the unnumbered equation on the bottom of page 312 (Mielke).
3269     s_lower = S[js - 1] + 1
3270     s_upper = S[js] + 1
3271     phi_J = [np.arange(s_lower[idx], s_upper[idx]) for idx in range(k)]
3272 
3273     # for every range in the above array, determine the sum of psi(I) for
3274     # every element in the range. Divide all the sums by `t`. Following the
3275     # last unnumbered equation on page 312.
3276     phis = [np.sum(psi(I_j)) for I_j in phi_J] / t[js]
3277 
3278     # `T` is equal to a[j] * phi[j], per the first unnumbered equation on
3279     # page 312. `phis` is already in the order based on `js`, so we index
3280     # into `a` with `js` as well.
3281     T = sum(phis * a[js])
3282

3284     E_0_T = n * (N * N - 1) / 12
3285 
3286     varM = (m * n * (N + 1.0) * (N ** 2 - 4) / 180 -
3287             m * n / (180 * N * (N - 1)) * np.sum(
3288                 t * (t**2 - 1) * (t**2 - 4 + (15 * (N - S - S_i_m1) ** 2))
3289             ))
3290 
3291     return ((T - E_0_T) / np.sqrt(varM),)

3390                        pvalue=array([8.32505043e-09, 8.98287869e-10]))

3402                                 ax != axis])):

3404                          "should match")

3445         pval.shape = res_shape

3479 def wilcoxon(x, y=None, zero_method="wilcox", correction=False,
3480              alternative="two-sided", method='auto', *, axis=0):
3481     """Calculate the Wilcoxon signed-rank test. 
3482  
3483     The Wilcoxon signed-rank test tests the null hypothesis that two 
3484     related paired samples come from the same distribution. In particular, 
3485     it tests whether the distribution of the differences ``x - y`` is symmetric 
3486     about zero. It is a non-parametric version of the paired T-test. 
3487  
3488     Parameters 
3489     ---------- 
3490     x : array_like 
3491         Either the first set of measurements (in which case ``y`` is the second 
3492         set of measurements), or the differences between two sets of 
3493         measurements (in which case ``y`` is not to be specified.)  Must be 
3494         one-dimensional. 
3495     y : array_like, optional 
3496         Either the second set of measurements (if ``x`` is the first set of 
3497         measurements), or not specified (if ``x`` is the differences between 
3498         two sets of measurements.)  Must be one-dimensional. 
3499  
3500         .. warning:: 
3501             When `y` is provided, `wilcoxon` calculates the test statistic 
3502             based on the ranks of the absolute values of ``d = x - y``. 
3503             Roundoff error in the subtraction can result in elements of ``d`` 
3504             being assigned different ranks even when they would be tied with 
3505             exact arithmetic. Rather than passing `x` and `y` separately, 
3506             consider computing the difference ``x - y``, rounding as needed to 
3507             ensure that only truly unique elements are numerically distinct, 
3508             and passing the result as `x`, leaving `y` at the default (None). 
3509  
3510     zero_method : {"wilcox", "pratt", "zsplit"}, optional 
3511         There are different conventions for handling pairs of observations 
3512         with equal values ("zero-differences", or "zeros"). 
3513  
3514         * "wilcox": Discards all zero-differences (default); see [4]_. 
3515         * "pratt": Includes zero-differences in the ranking process, 
3516           but drops the ranks of the zeros (more conservative); see [3]_. 
3517           In this case, the normal approximation is adjusted as in [5]_. 
3518         * "zsplit": Includes zero-differences in the ranking process and 
3519           splits the zero rank between positive and negative ones. 
3520  
3521     correction : bool, optional 
3522         If True, apply continuity correction by adjusting the Wilcoxon rank 
3523         statistic by 0.5 towards the mean value when computing the 
3524         z-statistic if a normal approximation is used.  Default is False. 
3525     alternative : {"two-sided", "greater", "less"}, optional 
3526         Defines the alternative hypothesis. Default is 'two-sided'. 
3527         In the following, let ``d`` represent the difference between the paired 
3528         samples: ``d = x - y`` if both ``x`` and ``y`` are provided, or 
3529         ``d = x`` otherwise. 
3530  
3531         * 'two-sided': the distribution underlying ``d`` is not symmetric 
3532           about zero. 
3533         * 'less': the distribution underlying ``d`` is stochastically less 
3534           than a distribution symmetric about zero. 
3535         * 'greater': the distribution underlying ``d`` is stochastically 
3536           greater than a distribution symmetric about zero. 
3537  
3538     method : {"auto", "exact", "asymptotic"} or `PermutationMethod` instance, optional 
3539         Method to calculate the p-value, see Notes. Default is "auto". 
3540  
3541     axis : int or None, default: 0 
3542         If an int, the axis of the input along which to compute the statistic. 
3543         The statistic of each axis-slice (e.g. row) of the input will appear 
3544         in a corresponding element of the output. If ``None``, the input will 
3545         be raveled before computing the statistic. 
3546  
3547     Returns 
3548     ------- 
3549     An object with the following attributes. 
3550  
3551     statistic : array_like 
3552         If `alternative` is "two-sided", the sum of the ranks of the 
3553         differences above or below zero, whichever is smaller. 
3554         Otherwise the sum of the ranks of the differences above zero. 
3555     pvalue : array_like 
3556         The p-value for the test depending on `alternative` and `method`. 
3557     zstatistic : array_like 
3558         When ``method = 'asymptotic'``, this is the normalized z-statistic:: 
3559  
3560             z = (T - mn - d) / se 
3561  
3562         where ``T`` is `statistic` as defined above, ``mn`` is the mean of the 
3563         distribution under the null hypothesis, ``d`` is a continuity 
3564         correction, and ``se`` is the standard error. 
3565         When ``method != 'asymptotic'``, this attribute is not available. 
3566  
3567     See Also 
3568     -------- 
3569     kruskal, mannwhitneyu 
3570  
3571     Notes 
3572     ----- 
3573     In the following, let ``d`` represent the difference between the paired 
3574     samples: ``d = x - y`` if both ``x`` and ``y`` are provided, or ``d = x`` 
3575     otherwise. Assume that all elements of ``d`` are independent and 
3576     identically distributed observations, and all are distinct and nonzero. 
3577  
3578     - When ``len(d)`` is sufficiently large, the null distribution of the 
3579       normalized test statistic (`zstatistic` above) is approximately normal, 
3580       and ``method = 'asymptotic'`` can be used to compute the p-value. 
3581  
3582     - When ``len(d)`` is small, the normal approximation may not be accurate, 
3583       and ``method='exact'`` is preferred (at the cost of additional 
3584       execution time). 
3585  
3586     - The default, ``method='auto'``, selects between the two: 
3587       ``method='exact'`` is used when ``len(d) <= 50``, and 
3588       ``method='asymptotic'`` is used otherwise. 
3589  
3590     The presence of "ties" (i.e. not all elements of ``d`` are unique) or 
3591     "zeros" (i.e. elements of ``d`` are zero) changes the null distribution 
3592     of the test statistic, and ``method='exact'`` no longer calculates 
3593     the exact p-value. If ``method='asymptotic'``, the z-statistic is adjusted 
3594     for more accurate comparison against the standard normal, but still, 
3595     for finite sample sizes, the standard normal is only an approximation of 
3596     the true null distribution of the z-statistic. For such situations, the 
3597     `method` parameter also accepts instances of `PermutationMethod`. In this 
3598     case, the p-value is computed using `permutation_test` with the provided 
3599     configuration options and other appropriate settings. 
3600  
3601     The presence of ties and zeros affects the resolution of ``method='auto'`` 
3602     accordingly: exhasutive permutations are performed when ``len(d) <= 13``, 
3603     and the asymptotic method is used otherwise. Note that they asymptotic 
3604     method may not be very accurate even for ``len(d) > 14``; the threshold 
3605     was chosen as a compromise between execution time and accuracy under the 
3606     constraint that the results must be deterministic. Consider providing an 
3607     instance of `PermutationMethod` method manually, choosing the 
3608     ``n_resamples`` parameter to balance time constraints and accuracy 
3609     requirements. 
3610  
3611     Please also note that in the edge case that all elements of ``d`` are zero, 
3612     the p-value relying on the normal approximaton cannot be computed (NaN) 
3613     if ``zero_method='wilcox'`` or ``zero_method='pratt'``. 
3614  
3615     References 
3616     ---------- 
3617     .. [1] https://en.wikipedia.org/wiki/Wilcoxon_signed-rank_test 
3618     .. [2] Conover, W.J., Practical Nonparametric Statistics, 1971. 
3619     .. [3] Pratt, J.W., Remarks on Zeros and Ties in the Wilcoxon Signed 
3620        Rank Procedures, Journal of the American Statistical Association, 
3621        Vol. 54, 1959, pp. 655-667. :doi:`10.1080/01621459.1959.10501526` 
3622     .. [4] Wilcoxon, F., Individual Comparisons by Ranking Methods,

3629     Examples 
3630     -------- 
3631     In [4]_, the differences in height between cross- and self-fertilized 
3632     corn plants is given as follows: 
3633  
3634     >>> d = [6, 8, 14, 16, 23, 24, 28, 29, 41, -48, 49, 56, 60, -67, 75] 
3635  
3636     Cross-fertilized plants appear to be higher. To test the null 
3637     hypothesis that there is no height difference, we can apply the 
3638     two-sided test: 
3639  
3640     >>> from scipy.stats import wilcoxon 
3641     >>> res = wilcoxon(d) 
3642     >>> res.statistic, res.pvalue 
3643     (24.0, 0.041259765625) 
3644  
3645     Hence, we would reject the null hypothesis at a confidence level of 5%, 
3646     concluding that there is a difference in height between the groups. 
3647     To confirm that the median of the differences can be assumed to be 
3648     positive, we use: 
3649  
3650     >>> res = wilcoxon(d, alternative='greater') 
3651     >>> res.statistic, res.pvalue 
3652     (96.0, 0.0206298828125) 
3653  
3654     This shows that the null hypothesis that the median is negative can be 
3655     rejected at a confidence level of 5% in favor of the alternative that 
3656     the median is greater than zero. The p-values above are exact. Using the 
3657     normal approximation gives very similar values: 
3658  
3659     >>> res = wilcoxon(d, method='asymptotic') 
3660     >>> res.statistic, res.pvalue 
3661     (24.0, 0.04088813291185591) 
3662  
3663     Note that the statistic changed to 96 in the one-sided case (the sum 
3664     of ranks of positive differences) whereas it is 24 in the two-sided 
3665     case (the minimum of sum of ranks above and below zero). 
3666  
3667     In the example above, the differences in height between paired plants are 
3668     provided to `wilcoxon` directly. Alternatively, `wilcoxon` accepts two 
3669     samples of equal length, calculates the differences between paired 
3670     elements, then performs the test. Consider the samples ``x`` and ``y``: 
3671  
3672     >>> import numpy as np 
3673     >>> x = np.array([0.5, 0.825, 0.375, 0.5]) 
3674     >>> y = np.array([0.525, 0.775, 0.325, 0.55]) 
3675     >>> res = wilcoxon(x, y, alternative='greater') 
3676     >>> res 
3677     WilcoxonResult(statistic=5.0, pvalue=0.5625) 
3678  
3679     Note that had we calculated the differences by hand, the test would have 
3680     produced different results: 
3681  
3682     >>> d = [-0.025, 0.05, 0.05, -0.05] 
3683     >>> ref = wilcoxon(d, alternative='greater') 
3684     >>> ref 
3685     WilcoxonResult(statistic=6.0, pvalue=0.5) 
3686  
3687     The substantial difference is due to roundoff error in the results of 
3688     ``x-y``: 
3689  
3690     >>> d - (x-y) 
3691     array([2.08166817e-17, 6.93889390e-17, 1.38777878e-17, 4.16333634e-17]) 
3692  
3693     Even though we expected all the elements of ``(x-y)[1:]`` to have the same 
3694     magnitude ``0.05``, they have slightly different magnitudes in practice, 
3695     and therefore are assigned different ranks in the test. Before performing 
3696     the test, consider calculating ``d`` and adjusting it as necessary to 
3697     ensure that theoretically identically values are not numerically distinct. 
3698     For example: 
3699  
3700     >>> d2 = np.around(x - y, decimals=3) 
3701     >>> wilcoxon(d2, alternative='greater') 
3702     WilcoxonResult(statistic=6.0, pvalue=0.5) 
3703  
3704     """

3706     if method == "approx":
3707         method = "asymptotic"
3708     return _wilcoxon._wilcoxon_nd(x, y, zero_method, correction, alternative,
3709                                   method, axis)

3718 def median_test(*samples, ties='below', correction=True, lambda_=1,
3719                 nan_policy='propagate'):
3720     """Perform a Mood's median test. 
3721  
3722     Test that two or more samples come from populations with the same median. 
3723  
3724     Let ``n = len(samples)`` be the number of samples.  The "grand median" of 
3725     all the data is computed, and a contingency table is formed by 
3726     classifying the values in each sample as being above or below the grand 
3727     median.  The contingency table, along with `correction` and `lambda_`, 
3728     are passed to `scipy.stats.chi2_contingency` to compute the test statistic 
3729     and p-value. 
3730  
3731     Parameters 
3732     ---------- 
3733     sample1, sample2, ... : array_like 
3734         The set of samples.  There must be at least two samples. 
3735         Each sample must be a one-dimensional sequence containing at least 
3736         one value.  The samples are not required to have the same length. 
3737     ties : str, optional 
3738         Determines how values equal to the grand median are classified in 
3739         the contingency table.  The string must be one of:: 
3740  
3741             "below": 
3742                 Values equal to the grand median are counted as "below". 
3743             "above": 
3744                 Values equal to the grand median are counted as "above". 
3745             "ignore": 
3746                 Values equal to the grand median are not counted. 
3747  
3748         The default is "below". 
3749     correction : bool, optional 
3750         If True, *and* there are just two samples, apply Yates' correction 
3751         for continuity when computing the test statistic associated with 
3752         the contingency table.  Default is True. 
3753     lambda_ : float or str, optional 
3754         By default, the statistic computed in this test is Pearson's 
3755         chi-squared statistic.  `lambda_` allows a statistic from the 
3756         Cressie-Read power divergence family to be used instead.  See 
3757         `power_divergence` for details. 
3758         Default is 1 (Pearson's chi-squared statistic). 
3759     nan_policy : {'propagate', 'raise', 'omit'}, optional 
3760         Defines how to handle when input contains nan. 'propagate' returns nan, 
3761         'raise' throws an error, 'omit' performs the calculations ignoring nan 
3762         values. Default is 'propagate'. 
3763  
3764     Returns 
3765     ------- 
3766     res : MedianTestResult 
3767         An object containing attributes: 
3768  
3769         statistic : float 
3770             The test statistic.  The statistic that is returned is determined 
3771             by `lambda_`.  The default is Pearson's chi-squared statistic. 
3772         pvalue : float 
3773             The p-value of the test. 
3774         median : float 
3775             The grand median. 
3776         table : ndarray 
3777             The contingency table.  The shape of the table is (2, n), where 
3778             n is the number of samples.  The first row holds the counts of the 
3779             values above the grand median, and the second row holds the counts 
3780             of the values below the grand median.  The table allows further 
3781             analysis with, for example, `scipy.stats.chi2_contingency`, or with 
3782             `scipy.stats.fisher_exact` if there are two samples, without having 
3783             to recompute the table.  If ``nan_policy`` is "propagate" and there 
3784             are nans in the input, the return value for ``table`` is ``None``. 
3785  
3786     See Also 
3787     -------- 
3788     kruskal : Compute the Kruskal-Wallis H-test for independent samples. 
3789     mannwhitneyu : Computes the Mann-Whitney rank test on samples x and y. 
3790  
3791     Notes 
3792     ----- 
3793     .. versionadded:: 0.15.0 
3794  
3795     References 
3796     ---------- 
3797     .. [1] Mood, A. M., Introduction to the Theory of Statistics. McGraw-Hill 
3798         (1950), pp. 394-399. 
3799     .. [2] Zar, J. H., Biostatistical Analysis, 5th ed. Prentice Hall (2010). 
3800         See Sections 8.12 and 10.15. 
3801  
3802     Examples 
3803     -------- 
3804     A biologist runs an experiment in which there are three groups of plants. 
3805     Group 1 has 16 plants, group 2 has 15 plants, and group 3 has 17 plants. 
3806     Each plant produces a number of seeds.  The seed counts for each group 
3807     are:: 
3808  
3809         Group 1: 10 14 14 18 20 22 24 25 31 31 32 39 43 43 48 49 
3810         Group 2: 28 30 31 33 34 35 36 40 44 55 57 61 91 92 99 
3811         Group 3:  0  3  9 22 23 25 25 33 34 34 40 45 46 48 62 67 84 
3812  
3813     The following code applies Mood's median test to these samples. 
3814  
3815     >>> g1 = [10, 14, 14, 18, 20, 22, 24, 25, 31, 31, 32, 39, 43, 43, 48, 49] 
3816     >>> g2 = [28, 30, 31, 33, 34, 35, 36, 40, 44, 55, 57, 61, 91, 92, 99] 
3817     >>> g3 = [0, 3, 9, 22, 23, 25, 25, 33, 34, 34, 40, 45, 46, 48, 62, 67, 84] 
3818     >>> from scipy.stats import median_test 
3819     >>> res = median_test(g1, g2, g3) 
3820  
3821     The median is 
3822  
3823     >>> res.median 
3824     34.0 
3825  
3826     and the contingency table is 
3827  
3828     >>> res.table 
3829     array([[ 5, 10,  7], 
3830            [11,  5, 10]]) 
3831  
3832     `p` is too large to conclude that the medians are not the same: 
3833  
3834     >>> res.pvalue 
3835     0.12609082774093244 
3836  
3837     The "G-test" can be performed by passing ``lambda_="log-likelihood"`` to 
3838     `median_test`. 
3839  
3840     >>> res = median_test(g1, g2, g3, lambda_="log-likelihood") 
3841     >>> res.pvalue 
3842     0.12224779737117837 
3843  
3844     The median occurs several times in the data, so we'll get a different 
3845     result if, for example, ``ties="above"`` is used: 
3846  
3847     >>> res = median_test(g1, g2, g3, ties="above") 
3848     >>> res.pvalue 
3849     0.063873276069553273 
3850  
3851     >>> res.table 
3852     array([[ 5, 11,  9], 
3853            [11,  4,  8]]) 
3854  
3855     This example demonstrates that if the data set is not large and there 
3856     are values equal to the median, the p-value can be sensitive to the 
3857     choice of `ties`. 
3858  
3859     """
3860     if len(samples) < 2:
3861         raise ValueError('median_test requires two or more samples.')
3862 
3863     ties_options = ['below', 'above', 'ignore']
3864     if ties not in ties_options:
3865         raise ValueError(f"invalid 'ties' option '{ties}'; 'ties' must be one "
3866                          f"of: {str(ties_options)[1:-1]}")
3867 
3868     data = [np.asarray(sample) for sample in samples]
3869 
3870     # Validate the sizes and shapes of the arguments.
3871     for k, d in enumerate(data):
3872         if d.size == 0:
3873             raise ValueError("Sample %d is empty. All samples must "
3874                              "contain at least one value." % (k + 1))
3875         if d.ndim != 1:
3876             raise ValueError("Sample %d has %d dimensions.  All "
3877                              "samples must be one-dimensional sequences." %
3878                              (k + 1, d.ndim))
3879 
3880     cdata = np.concatenate(data)
3881     contains_nan, nan_policy = _contains_nan(cdata, nan_policy)
3882     if contains_nan and nan_policy == 'propagate':
3883         return MedianTestResult(np.nan, np.nan, np.nan, None)
3884 
3885     if contains_nan:
3886         grand_median = np.median(cdata[~np.isnan(cdata)])
3887     else:
3888         grand_median = np.median(cdata)
3889     # When the minimum version of numpy supported by scipy is 1.9.0,
3890     # the above if/else statement can be replaced by the single line:
3891     #     grand_median = np.nanmedian(cdata)
3892 
3893     # Create the contingency table.
3894     table = np.zeros((2, len(data)), dtype=np.int64)
3895     for k, sample in enumerate(data):
3896         sample = sample[~np.isnan(sample)]
3897 
3898         nabove = count_nonzero(sample > grand_median)
3899         nbelow = count_nonzero(sample < grand_median)
3900         nequal = sample.size - (nabove + nbelow)
3901         table[0, k] += nabove
3902         table[1, k] += nbelow
3903         if ties == "below":
3904             table[1, k] += nequal
3905         elif ties == "above":
3906             table[0, k] += nequal
3907 
3908     # Check that no row or column of the table is all zero.
3909     # Such a table can not be given to chi2_contingency, because it would have
3910     # a zero in the table of expected frequencies.
3911     rowsums = table.sum(axis=1)
3912     if rowsums[0] == 0:
3913         raise ValueError(f"All values are below the grand median ({grand_median}).")
3914     if rowsums[1] == 0:
3915         raise ValueError(f"All values are above the grand median ({grand_median}).")
3916     if ties == "ignore":
3917         # We already checked that each sample has at least one value, but it
3918         # is possible that all those values equal the grand median.  If `ties`
3919         # is "ignore", that would result in a column of zeros in `table`.  We
3920         # check for that case here.
3921         zero_cols = np.nonzero((table == 0).all(axis=0))[0]
3922         if len(zero_cols) > 0:
3923             msg = ("All values in sample %d are equal to the grand "
3924                    "median (%r), so they are ignored, resulting in an "

3962  
3963     where :math:`i` is the imaginary unit and :math:`\mathop{\mathrm{Arg}} z` 
3964     gives the principal value of the argument of complex number :math:`z`,

3966     above expression is known as the `mean resultant vector`. 
3967

3969     ----------

3972         equal to ``(high - low)``. 
3973     high : float, optional 
3974         Upper boundary of the principal value of an angle.  Default is ``2*pi``. 
3975     low : float, optional 
3976         Lower boundary of the principal value of an angle.  Default is ``0``. 
3977  
3978     Returns 
3979     ------- 
3980     circmean : float 
3981         Circular mean, restricted to the range ``[low, high]``. 
3982  
3983         If the mean resultant vector is zero, an input-dependent, 
3984         implementation-defined number between ``[low, high]`` is returned. 
3985         If the input array is empty, ``np.nan`` is returned. 
3986

3988     -------- 
3989     circstd : Circular standard deviation. 
3990     circvar : Circular variance. 
3991  
3992     References 
3993     ---------- 
3994     .. [1] Mardia, K. V. and Jupp, P. E. *Directional Statistics*. 
3995            John Wiley & Sons, 1999. 
3996

4252 def directional_stats(samples, *, axis=0, normalize=True):
4253     """ 
4254     Computes sample statistics for directional data. 
4255  
4256     Computes the directional mean (also called the mean direction vector) and 
4257     mean resultant length of a sample of vectors. 
4258  
4259     The directional mean is a measure of "preferred direction" of vector data. 
4260     It is analogous to the sample mean, but it is for use when the length of 
4261     the data is irrelevant (e.g. unit vectors). 
4262  
4263     The mean resultant length is a value between 0 and 1 used to quantify the 
4264     dispersion of directional data: the smaller the mean resultant length, the 
4265     greater the dispersion. Several definitions of directional variance 
4266     involving the mean resultant length are given in [1]_ and [2]_. 
4267  
4268     Parameters 
4269     ---------- 
4270     samples : array_like 
4271         Input array. Must be at least two-dimensional, and the last axis of the 
4272         input must correspond with the dimensionality of the vector space. 
4273         When the input is exactly two dimensional, this means that each row 
4274         of the data is a vector observation. 
4275     axis : int, default: 0 
4276         Axis along which the directional mean is computed. 
4277     normalize: boolean, default: True 
4278         If True, normalize the input to ensure that each observation is a 
4279         unit vector. It the observations are already unit vectors, consider 
4280         setting this to False to avoid unnecessary computation. 
4281  
4282     Returns 
4283     ------- 
4284     res : DirectionalStats 
4285         An object containing attributes: 
4286  
4287         mean_direction : ndarray 
4288             Directional mean. 
4289         mean_resultant_length : ndarray 
4290             The mean resultant length [1]_. 
4291  
4292     See Also 
4293     -------- 
4294     circmean: circular mean; i.e. directional mean for 2D *angles* 
4295     circvar: circular variance; i.e. directional variance for 2D *angles* 
4296  
4297     Notes 
4298     ----- 
4299     This uses a definition of directional mean from [1]_. 
4300     Assuming the observations are unit vectors, the calculation is as follows. 
4301  
4302     .. code-block:: python 
4303  
4304         mean = samples.mean(axis=0) 
4305         mean_resultant_length = np.linalg.norm(mean) 
4306         mean_direction = mean / mean_resultant_length 
4307  
4308     This definition is appropriate for *directional* data (i.e. vector data 
4309     for which the magnitude of each observation is irrelevant) but not 
4310     for *axial* data (i.e. vector data for which the magnitude and *sign* of 
4311     each observation is irrelevant). 
4312  
4313     Several definitions of directional variance involving the mean resultant 
4314     length ``R`` have been proposed, including ``1 - R`` [1]_, ``1 - R**2`` 
4315     [2]_, and ``2 * (1 - R)`` [2]_. Rather than choosing one, this function 
4316     returns ``R`` as attribute `mean_resultant_length` so the user can compute 
4317     their preferred measure of dispersion. 
4318  
4319     References 
4320     ---------- 
4321     .. [1] Mardia, Jupp. (2000). *Directional Statistics* 
4322        (p. 163). Wiley. 
4323  
4324     .. [2] https://en.wikipedia.org/wiki/Directional_statistics 
4325  
4326     Examples 
4327     -------- 
4328     >>> import numpy as np 
4329     >>> from scipy.stats import directional_stats 
4330     >>> data = np.array([[3, 4],    # first observation, 2D vector space 
4331     ...                  [6, -8]])  # second observation 
4332     >>> dirstats = directional_stats(data) 
4333     >>> dirstats.mean_direction 
4334     array([1., 0.]) 
4335  
4336     In contrast, the regular sample mean of the vectors would be influenced 
4337     by the magnitude of each observation. Furthermore, the result would not be 
4338     a unit vector. 
4339  
4340     >>> data.mean(axis=0) 
4341     array([4.5, -2.]) 
4342  
4343     An exemplary use case for `directional_stats` is to find a *meaningful* 
4344     center for a set of observations on a sphere, e.g. geographical locations. 
4345  
4346     >>> data = np.array([[0.8660254, 0.5, 0.], 
4347     ...                  [0.8660254, -0.5, 0.]]) 
4348     >>> dirstats = directional_stats(data) 
4349     >>> dirstats.mean_direction 
4350     array([1., 0., 0.]) 
4351  
4352     The regular sample mean on the other hand yields a result which does not 
4353     lie on the surface of the sphere. 
4354  
4355     >>> data.mean(axis=0) 
4356     array([0.8660254, 0., 0.]) 
4357  
4358     The function also returns the mean resultant length, which 
4359     can be used to calculate a directional variance. For example, using the 
4360     definition ``Var(z) = 1 - R`` from [2]_ where ``R`` is the 
4361     mean resultant length, we can calculate the directional variance of the 
4362     vectors in the above example as: 
4363  
4364     >>> 1 - dirstats.mean_resultant_length 
4365     0.13397459716167093 
4366     """
4367     xp = array_namespace(samples)
4368     samples = xp.asarray(samples)
4369 
4370     if samples.ndim < 2:
4371         raise ValueError("samples must at least be two-dimensional. "
4372                          f"Instead samples has shape: {tuple(samples.shape)}")
4373     samples = xp.moveaxis(samples, axis, 0)
4374     if normalize:
4375         vectornorms = xp_vector_norm(samples, axis=-1, keepdims=True, xp=xp)
4376         samples = samples/vectornorms
4377     mean = xp.mean(samples, axis=0)
4378     mean_resultant_length = xp_vector_norm(mean, axis=-1, keepdims=True, xp=xp)
4379     mean_direction = mean / mean_resultant_length
4380     mrl = xp.squeeze(mean_resultant_length, axis=-1)
4381     mean_resultant_length = mrl[()] if mrl.ndim == 0 else mrl
4382     return DirectionalStats(mean_direction, mean_resultant_length)

4385 def false_discovery_control(ps, *, axis=0, method='bh'):
4386     """Adjust p-values to control the false discovery rate. 
4387  
4388     The false discovery rate (FDR) is the expected proportion of rejected null 
4389     hypotheses that are actually true. 
4390     If the null hypothesis is rejected when the *adjusted* p-value falls below 
4391     a specified level, the false discovery rate is controlled at that level. 
4392  
4393     Parameters 
4394     ---------- 
4395     ps : 1D array_like 
4396         The p-values to adjust. Elements must be real numbers between 0 and 1. 
4397     axis : int 
4398         The axis along which to perform the adjustment. The adjustment is 
4399         performed independently along each axis-slice. If `axis` is None, `ps` 
4400         is raveled before performing the adjustment. 
4401     method : {'bh', 'by'} 
4402         The false discovery rate control procedure to apply: ``'bh'`` is for 
4403         Benjamini-Hochberg [1]_ (Eq. 1), ``'by'`` is for Benjaminini-Yekutieli 
4404         [2]_ (Theorem 1.3). The latter is more conservative, but it is 
4405         guaranteed to control the FDR even when the p-values are not from 
4406         independent tests. 
4407  
4408     Returns 
4409     ------- 
4410     ps_adusted : array_like 
4411         The adjusted p-values. If the null hypothesis is rejected where these 
4412         fall below a specified level, the false discovery rate is controlled 
4413         at that level. 
4414  
4415     See Also 
4416     -------- 
4417     combine_pvalues 
4418     statsmodels.stats.multitest.multipletests 
4419  
4420     Notes 
4421     ----- 
4422     In multiple hypothesis testing, false discovery control procedures tend to 
4423     offer higher power than familywise error rate control procedures (e.g. 
4424     Bonferroni correction [1]_). 
4425  
4426     If the p-values correspond with independent tests (or tests with 
4427     "positive regression dependencies" [2]_), rejecting null hypotheses 
4428     corresponding with Benjamini-Hochberg-adjusted p-values below :math:`q` 
4429     controls the false discovery rate at a level less than or equal to 
4430     :math:`q m_0 / m`, where :math:`m_0` is the number of true null hypotheses 
4431     and :math:`m` is the total number of null hypotheses tested. The same is 
4432     true even for dependent tests when the p-values are adjusted accorded to 
4433     the more conservative Benjaminini-Yekutieli procedure. 
4434  
4435     The adjusted p-values produced by this function are comparable to those 
4436     produced by the R function ``p.adjust`` and the statsmodels function 
4437     `statsmodels.stats.multitest.multipletests`. Please consider the latter 
4438     for more advanced methods of multiple comparison correction. 
4439  
4440     References 
4441     ---------- 
4442     .. [1] Benjamini, Yoav, and Yosef Hochberg. "Controlling the false 
4443            discovery rate: a practical and powerful approach to multiple 
4444            testing." Journal of the Royal statistical society: series B 
4445            (Methodological) 57.1 (1995): 289-300. 
4446  
4447     .. [2] Benjamini, Yoav, and Daniel Yekutieli. "The control of the false 
4448            discovery rate in multiple testing under dependency." Annals of 
4449            statistics (2001): 1165-1188. 
4450  
4451     .. [3] TileStats. FDR - Benjamini-Hochberg explained - Youtube. 
4452            https://www.youtube.com/watch?v=rZKa4tW2NKs. 
4453  
4454     .. [4] Neuhaus, Karl-Ludwig, et al. "Improved thrombolysis in acute 
4455            myocardial infarction with front-loaded administration of alteplase: 
4456            results of the rt-PA-APSAC patency study (TAPS)." Journal of the 
4457            American College of Cardiology 19.5 (1992): 885-891. 
4458  
4459     Examples 
4460     -------- 
4461     We follow the example from [1]_. 
4462  
4463         Thrombolysis with recombinant tissue-type plasminogen activator (rt-PA) 
4464         and anisoylated plasminogen streptokinase activator (APSAC) in 
4465         myocardial infarction has been proved to reduce mortality. [4]_ 
4466         investigated the effects of a new front-loaded administration of rt-PA 
4467         versus those obtained with a standard regimen of APSAC, in a randomized
