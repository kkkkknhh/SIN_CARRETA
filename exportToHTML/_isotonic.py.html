15   def isotonic_regression(
16       y: "npt.ArrayLike",
17       *,
18       weights: "npt.ArrayLike | None" = None,
19       increasing: bool = True,
20   ) -> OptimizeResult:
21       r"""Nonparametric isotonic regression. 
22    
23       A (not strictly) monotonically increasing array `x` with the same length 
24       as `y` is calculated by the pool adjacent violators algorithm (PAVA), see 
25       [1]_. See the Notes section for more details. 
26    
27       Parameters 
28       ---------- 
29       y : (N,) array_like 
30           Response variable. 
31       weights : (N,) array_like or None 
32           Case weights. 
33       increasing : bool 
34           If True, fit monotonic increasing, i.e. isotonic, regression. 
35           If False, fit a monotonic decreasing, i.e. antitonic, regression. 
36           Default is True. 
37    
38       Returns 
39       ------- 
40       res : OptimizeResult 
41           The optimization result represented as a ``OptimizeResult`` object. 
42           Important attributes are: 
43    
44           - ``x``: The isotonic regression solution, i.e. an increasing (or 
45             decreasing) array of the same length than y, with elements in the 
46             range from min(y) to max(y). 
47           - ``weights`` : Array with the sum of case weights for each block 
48             (or pool) B. 
49           - ``blocks``: Array of length B+1 with the indices of the start 
50             positions of each block (or pool) B. The j-th block is given by 
51             ``x[blocks[j]:blocks[j+1]]`` for which all values are the same. 
52    
53       Notes 
54       ----- 
55       Given data :math:`y` and case weights :math:`w`, the isotonic regression 
56       solves the following optimization problem: 
57    
58       .. math:: 
59    
60           \operatorname{argmin}_{x_i} \sum_i w_i (y_i - x_i)^2 \quad 
61           \text{subject to } x_i \leq x_j \text{ whenever } i \leq j \,. 
62    
63       For every input value :math:`y_i`, it generates a value :math:`x_i` such 
64       that :math:`x` is increasing (but not strictly), i.e. 
65       :math:`x_i \leq x_{i+1}`. This is accomplished by the PAVA. 
66       The solution consists of pools or blocks, i.e. neighboring elements of 
67       :math:`x`, e.g. :math:`x_i` and :math:`x_{i+1}`, that all have the same 
68       value. 
69    
70       Most interestingly, the solution stays the same if the squared loss is 
71       replaced by the wide class of Bregman functions which are the unique 
72       class of strictly consistent scoring functions for the mean, see [2]_ 
73       and references therein. 
74    
75       The implemented version of PAVA according to [1]_ has a computational 
76       complexity of O(N) with input size N. 
77    
78       References 
79       ---------- 
80       .. [1] Busing, F. M. T. A. (2022). 
81              Monotone Regression: A Simple and Fast O(n) PAVA Implementation. 
82              Journal of Statistical Software, Code Snippets, 102(1), 1-25. 
83              :doi:`10.18637/jss.v102.c01` 
84       .. [2] Jordan, A.I., MÃ¼hlemann, A. & Ziegel, J.F. 
85              Characterizing the optimal solutions to the isotonic regression 
86              problem for identifiable functionals. 
87              Ann Inst Stat Math 74, 489-514 (2022). 
88              :doi:`10.1007/s10463-021-00808-0` 
89    
90       Examples 
91       -------- 
92       This example demonstrates that ``isotonic_regression`` really solves a 
93       constrained optimization problem. 
94    
95       >>> import numpy as np 
96       >>> from scipy.optimize import isotonic_regression, minimize 
97       >>> y = [1.5, 1.0, 4.0, 6.0, 5.7, 5.0, 7.8, 9.0, 7.5, 9.5, 9.0] 
98       >>> def objective(yhat, y): 
99       ...     return np.sum((yhat - y)**2) 
100      >>> def constraint(yhat, y): 
101      ...     # This is for a monotonically increasing regression. 
102      ...     return np.diff(yhat) 
103      >>> result = minimize(objective, x0=y, args=(y,), 
104      ...                   constraints=[{'type': 'ineq', 
105      ...                                 'fun': lambda x: constraint(x, y)}]) 
106      >>> result.x 
107      array([1.25      , 1.25      , 4.        , 5.56666667, 5.56666667, 
108             5.56666667, 7.8       , 8.25      , 8.25      , 9.25      , 
109             9.25      ]) 
110      >>> result = isotonic_regression(y) 
111      >>> result.x 
112      array([1.25      , 1.25      , 4.        , 5.56666667, 5.56666667, 
113             5.56666667, 7.8       , 8.25      , 8.25      , 9.25      , 
114             9.25      ]) 
115   
116      The big advantage of ``isotonic_regression`` compared to calling 
117      ``minimize`` is that it is more user friendly, i.e. one does not need to 
118      define objective and constraint functions, and that it is orders of 
119      magnitudes faster. On commodity hardware (in 2023), for normal distributed 
120      input y of length 1000, the minimizer takes about 4 seconds, while 
121      ``isotonic_regression`` takes about 200 microseconds. 
122      """
123      yarr = np.atleast_1d(y)  # Check yarr.ndim == 1 is implicit (pybind11) in pava.
124      order = slice(None) if increasing else slice(None, None, -1)
125      x = np.array(yarr[order], order="C", dtype=np.float64, copy=True)
126      if weights is None:
127          wx = np.ones_like(yarr, dtype=np.float64)
128      else:
129          warr = np.atleast_1d(weights)
130  
131          if not (yarr.ndim == warr.ndim == 1 and yarr.shape[0] == warr.shape[0]):
132              raise ValueError(
133                  "Input arrays y and w must have one dimension of equal length."
134              )
135          if np.any(warr <= 0):
136              raise ValueError("Weights w must be strictly positive.")
137  
138          wx = np.array(warr[order], order="C", dtype=np.float64, copy=True)
139      n = x.shape[0]
140      r = np.full(shape=n + 1, fill_value=-1, dtype=np.intp)
141      x, wx, r, b = pava(x, wx, r)
142      # Now that we know the number of blocks b, we only keep the relevant part
143      # of r and wx.
144      # As information: Due to the pava implementation, after the last block
145      # index, there might be smaller numbers appended to r, e.g.
146      # r = [0, 10, 8, 7] which in the end should be r = [0, 10].
147      r = r[:b + 1]  # type: ignore[assignment]
148      wx = wx[:b]
149      if not increasing:
150          x = x[::-1]
151          wx = wx[::-1]
152          r = r[-1] - r[::-1]
153      return OptimizeResult(
154          x=x,
155          weights=wx,
156          blocks=r,
157      )
