19   class KanesMethod(_Methods):
20       r"""Kane's method object. 
21    
22       Explanation 
23       =========== 
24    
25       This object is used to do the "book-keeping" as you go through and form 
26       equations of motion in the way Kane presents in: 
27       Kane, T., Levinson, D. Dynamics Theory and Applications. 1985 McGraw-Hill 
28    
29       The attributes are for equations in the form [M] udot = forcing. 
30    
31       Attributes 
32       ========== 
33    
34       q, u : Matrix 
35           Matrices of the generalized coordinates and speeds 
36       bodies : iterable 
37           Iterable of Particle and RigidBody objects in the system. 
38       loads : iterable 
39           Iterable of (Point, vector) or (ReferenceFrame, vector) tuples 
40           describing the forces on the system. 
41       auxiliary_eqs : Matrix 
42           If applicable, the set of auxiliary Kane's 
43           equations used to solve for non-contributing 
44           forces. 
45       mass_matrix : Matrix 
46           The system's dynamics mass matrix: [k_d; k_dnh] 
47       forcing : Matrix 
48           The system's dynamics forcing vector: -[f_d; f_dnh] 
49       mass_matrix_kin : Matrix 
50           The "mass matrix" for kinematic differential equations: k_kqdot 
51       forcing_kin : Matrix 
52           The forcing vector for kinematic differential equations: -(k_ku*u + f_k) 
53       mass_matrix_full : Matrix 
54           The "mass matrix" for the u's and q's with dynamics and kinematics 
55       forcing_full : Matrix 
56           The "forcing vector" for the u's and q's with dynamics and kinematics 
57    
58       Parameters 
59       ========== 
60    
61       frame : ReferenceFrame 
62           The inertial reference frame for the system. 
63       q_ind : iterable of dynamicsymbols 
64           Independent generalized coordinates. 
65       u_ind : iterable of dynamicsymbols 
66           Independent generalized speeds. 
67       kd_eqs : iterable of Expr, optional 
68           Kinematic differential equations, which linearly relate the generalized 
69           speeds to the time-derivatives of the generalized coordinates. 
70       q_dependent : iterable of dynamicsymbols, optional 
71           Dependent generalized coordinates. 
72       configuration_constraints : iterable of Expr, optional 
73           Constraints on the system's configuration, i.e. holonomic constraints. 
74       u_dependent : iterable of dynamicsymbols, optional 
75           Dependent generalized speeds. 
76       velocity_constraints : iterable of Expr, optional 
77           Constraints on the system's velocity, i.e. the combination of the 
78           nonholonomic constraints and the time-derivative of the holonomic 
79           constraints. 
80       acceleration_constraints : iterable of Expr, optional 
81           Constraints on the system's acceleration, by default these are the 
82           time-derivative of the velocity constraints. 
83       u_auxiliary : iterable of dynamicsymbols, optional 
84           Auxiliary generalized speeds. 
85       bodies : iterable of Particle and/or RigidBody, optional 
86           The particles and rigid bodies in the system. 
87       forcelist : iterable of tuple[Point | ReferenceFrame, Vector], optional 
88           Forces and torques applied on the system. 
89       explicit_kinematics : bool 
90           Boolean whether the mass matrices and forcing vectors should use the 
91           explicit form (default) or implicit form for kinematics. 
92           See the notes for more details. 
93       kd_eqs_solver : str, callable 
94           Method used to solve the kinematic differential equations. If a string 
95           is supplied, it should be a valid method that can be used with the 
96           :meth:`sympy.matrices.matrixbase.MatrixBase.solve`. If a callable is 
97           supplied, it should have the format ``f(A, rhs)``, where it solves the 
98           equations and returns the solution. The default utilizes LU solve. See 
99           the notes for more information. 
100      constraint_solver : str, callable 
101          Method used to solve the velocity constraints. If a string is 
102          supplied, it should be a valid method that can be used with the 
103          :meth:`sympy.matrices.matrixbase.MatrixBase.solve`. If a callable is 
104          supplied, it should have the format ``f(A, rhs)``, where it solves the 
105          equations and returns the solution. The default utilizes LU solve. See 
106          the notes for more information. 
107   
108      Notes 
109      ===== 
110   
111      The mass matrices and forcing vectors related to kinematic equations 
112      are given in the explicit form by default. In other words, the kinematic 
113      mass matrix is $\mathbf{k_{k\dot{q}}} = \mathbf{I}$. 
114      In order to get the implicit form of those matrices/vectors, you can set the 
115      ``explicit_kinematics`` attribute to ``False``. So $\mathbf{k_{k\dot{q}}}$ 
116      is not necessarily an identity matrix. This can provide more compact 
117      equations for non-simple kinematics. 
118   
119      Two linear solvers can be supplied to ``KanesMethod``: one for solving the 
120      kinematic differential equations and one to solve the velocity constraints. 
121      Both of these sets of equations can be expressed as a linear system ``Ax = rhs``, 
122      which have to be solved in order to obtain the equations of motion. 
123   
124      The default solver ``'LU'``, which stands for LU solve, results relatively low 
125      number of operations. The weakness of this method is that it can result in zero 
126      division errors. 
127   
128      If zero divisions are encountered, a possible solver which may solve the problem 
129      is ``"CRAMER"``. This method uses Cramer's rule to solve the system. This method 
130      is slower and results in more operations than the default solver. However it only 
131      uses a single division by default per entry of the solution. 
132   
133      While a valid list of solvers can be found at 
134      :meth:`sympy.matrices.matrixbase.MatrixBase.solve`, it is also possible to supply a 
135      `callable`. This way it is possible to use a different solver routine. If the 
136      kinematic differential equations are not too complex it can be worth it to simplify 
137      the solution by using ``lambda A, b: simplify(Matrix.LUsolve(A, b))``. Another 
138      option solver one may use is :func:`sympy.solvers.solveset.linsolve`. This can be 
139      done using `lambda A, b: tuple(linsolve((A, b)))[0]`, where we select the first 
140      solution as our system should have only one unique solution. 
141   
142      Examples 
143      ======== 
144   
145      This is a simple example for a one degree of freedom translational 
146      spring-mass-damper. 
147   
148      In this example, we first need to do the kinematics. 
149      This involves creating generalized speeds and coordinates and their 
150      derivatives. 
151      Then we create a point and set its velocity in a frame. 
152   
153          >>> from sympy import symbols 
154          >>> from sympy.physics.mechanics import dynamicsymbols, ReferenceFrame 
155          >>> from sympy.physics.mechanics import Point, Particle, KanesMethod 
156          >>> q, u = dynamicsymbols('q u') 
157          >>> qd, ud = dynamicsymbols('q u', 1) 
158          >>> m, c, k = symbols('m c k') 
159          >>> N = ReferenceFrame('N') 
160          >>> P = Point('P') 
161          >>> P.set_vel(N, u * N.x) 
162   
163      Next we need to arrange/store information in the way that KanesMethod 
164      requires. The kinematic differential equations should be an iterable of 
165      expressions. A list of forces/torques must be constructed, where each entry 
166      in the list is a (Point, Vector) or (ReferenceFrame, Vector) tuple, where 
167      the Vectors represent the Force or Torque. 
168      Next a particle needs to be created, and it needs to have a point and mass 
169      assigned to it. 
170      Finally, a list of all bodies and particles needs to be created. 
171   
172          >>> kd = [qd - u] 
173          >>> FL = [(P, (-k * q - c * u) * N.x)] 
174          >>> pa = Particle('pa', P, m) 
175          >>> BL = [pa] 
176   
177      Finally we can generate the equations of motion. 
178      First we create the KanesMethod object and supply an inertial frame, 
179      coordinates, generalized speeds, and the kinematic differential equations. 
180      Additional quantities such as configuration and motion constraints, 
181      dependent coordinates and speeds, and auxiliary speeds are also supplied 
182      here (see the online documentation). 
183      Next we form FR* and FR to complete: Fr + Fr* = 0. 
184      We have the equations of motion at this point. 
185      It makes sense to rearrange them though, so we calculate the mass matrix and 
186      the forcing terms, for E.o.M. in the form: [MM] udot = forcing, where MM is 
187      the mass matrix, udot is a vector of the time derivatives of the 
188      generalized speeds, and forcing is a vector representing "forcing" terms. 
189   
190          >>> KM = KanesMethod(N, q_ind=[q], u_ind=[u], kd_eqs=kd) 
191          >>> (fr, frstar) = KM.kanes_equations(BL, FL) 
192          >>> MM = KM.mass_matrix 
193          >>> forcing = KM.forcing 
194          >>> rhs = MM.inv() * forcing 
195          >>> rhs 
196          Matrix([[(-c*u(t) - k*q(t))/m]]) 
197          >>> KM.linearize(A_and_B=True)[0] 
198          Matrix([ 
199          [   0,    1], 
200          [-k/m, -c/m]]) 
201   
202      Please look at the documentation pages for more information on how to 
203      perform linearization and how to deal with dependent coordinates & speeds, 
204      and how do deal with bringing non-contributing forces into evidence. 
205   
206      """
207  
208      def __init__(self, frame, q_ind, u_ind, kd_eqs=None, q_dependent=None,
209                   configuration_constraints=None, u_dependent=None,
210                   velocity_constraints=None, acceleration_constraints=None,
211                   u_auxiliary=None, bodies=None, forcelist=None,
212                   explicit_kinematics=True, kd_eqs_solver='LU',
213                   constraint_solver='LU'):
214  
215          """Please read the online documentation. """
216          if not q_ind:
217              q_ind = [dynamicsymbols('dummy_q')]
218              kd_eqs = [dynamicsymbols('dummy_kd')]
219  
220          if not isinstance(frame, ReferenceFrame):
221              raise TypeError('An inertial ReferenceFrame must be supplied')
222          self._inertial = frame
223  
224          self._fr = None
225          self._frstar = None
226  
227          self._forcelist = forcelist
228          self._bodylist = bodies
229  
230          self.explicit_kinematics = explicit_kinematics
231          self._constraint_solver = constraint_solver
232          self._initialize_vectors(q_ind, q_dependent, u_ind, u_dependent,
233                  u_auxiliary)
234          _validate_coordinates(self.q, self.u)
235          self._initialize_kindiffeq_matrices(kd_eqs, kd_eqs_solver)
236          self._initialize_constraint_matrices(
237              configuration_constraints, velocity_constraints,
238              acceleration_constraints, constraint_solver)

240      def _initialize_vectors(self, q_ind, q_dep, u_ind, u_dep, u_aux):
241          """Initialize the coordinate and speed vectors."""
242  
243          none_handler = lambda x: Matrix(x) if x else Matrix()
244  
245          # Initialize generalized coordinates
246          q_dep = none_handler(q_dep)
247          if not iterable(q_ind):
248              raise TypeError('Generalized coordinates must be an iterable.')
249          if not iterable(q_dep):
250              raise TypeError('Dependent coordinates must be an iterable.')
251          q_ind = Matrix(q_ind)
252          self._qdep = q_dep
253          self._q = Matrix([q_ind, q_dep])
254          self._qdot = self.q.diff(dynamicsymbols._t)
255  
256          # Initialize generalized speeds
257          u_dep = none_handler(u_dep)
258          if not iterable(u_ind):
259              raise TypeError('Generalized speeds must be an iterable.')
260          if not iterable(u_dep):
261              raise TypeError('Dependent speeds must be an iterable.')
262          u_ind = Matrix(u_ind)
263          self._udep = u_dep
264          self._u = Matrix([u_ind, u_dep])
265          self._udot = self.u.diff(dynamicsymbols._t)
266          self._uaux = none_handler(u_aux)

268      def _initialize_constraint_matrices(self, config, vel, acc, linear_solver='LU'):
269          """Initializes constraint matrices."""
270          linear_solver = _parse_linear_solver(linear_solver)
271          # Define vector dimensions
272          o = len(self.u)
273          m = len(self._udep)
274          p = o - m
275          none_handler = lambda x: Matrix(x) if x else Matrix()
276  
277          # Initialize configuration constraints
278          config = none_handler(config)
279          if len(self._qdep) != len(config):
280              raise ValueError('There must be an equal number of dependent '
281                               'coordinates and configuration constraints.')
282          self._f_h = none_handler(config)
283  
284          # Initialize velocity and acceleration constraints
285          vel = none_handler(vel)
286          acc = none_handler(acc)
287          if len(vel) != m:
288              raise ValueError('There must be an equal number of dependent '
289                               'speeds and velocity constraints.')
290          if acc and (len(acc) != m):
291              raise ValueError('There must be an equal number of dependent '
292                               'speeds and acceleration constraints.')
293          if vel:
294  
295              # When calling kanes_equations, another class instance will be
296              # created if auxiliary u's are present. In this case, the
297              # computation of kinetic differential equation matrices will be
298              # skipped as this was computed during the original KanesMethod
299              # object, and the qd_u_map will not be available.
300              if self._qdot_u_map is not None:
301                  vel = msubs(vel, self._qdot_u_map)
302              self._k_nh, f_nh_neg = linear_eq_to_matrix(vel, self.u[:])
303              self._f_nh = -f_nh_neg
304  
305              # If no acceleration constraints given, calculate them.
306              if not acc:
307                  _f_dnh = (self._k_nh.diff(dynamicsymbols._t) * self.u +
308                      self._f_nh.diff(dynamicsymbols._t))
309                  if self._qdot_u_map is not None:
310                      _f_dnh = msubs(_f_dnh, self._qdot_u_map)
311                  self._f_dnh = _f_dnh
312                  self._k_dnh = self._k_nh
313              else:
314                  if self._qdot_u_map is not None:
315                      acc = msubs(acc, self._qdot_u_map)
316  
317                  self._k_dnh, f_dnh_neg = linear_eq_to_matrix(acc, self._udot[:])
318                  self._f_dnh = -f_dnh_neg
319              # Form of non-holonomic constraints is B*u + C = 0.
320              # We partition B into independent and dependent columns:
321              # Ars is then -B_dep.inv() * B_ind, and it relates dependent speeds
322              # to independent speeds as: udep = Ars*uind, neglecting the C term.
323              B_ind = self._k_nh[:, :p]
324              B_dep = self._k_nh[:, p:o]
325              self._Ars = -linear_solver(B_dep, B_ind)
326          else:
327              self._f_nh = Matrix()
328              self._k_nh = Matrix()
329              self._f_dnh = Matrix()
330              self._k_dnh = Matrix()
331              self._Ars = Matrix()

333      def _initialize_kindiffeq_matrices(self, kdeqs, linear_solver='LU'):
334          """Initialize the kinematic differential equation matrices. 
335   
336          Parameters 
337          ========== 
338          kdeqs : sequence of sympy expressions 
339              Kinematic differential equations in the form of f(u,q',q,t) where 
340              f() = 0. The equations have to be linear in the time-derivatives of 
341              the generalized coordinates and in the generalized speeds. 
342   
343          """
344          linear_solver = _parse_linear_solver(linear_solver)
345          if kdeqs:
346              if len(self.q) != len(kdeqs):
347                  raise ValueError('There must be an equal number of kinematic '
348                                   'differential equations and coordinates.')
349  
350              u = self.u
351              qdot = self._qdot
352  
353              kdeqs = Matrix(kdeqs)
354  
355              u_zero = dict.fromkeys(u, 0)
356              uaux_zero = dict.fromkeys(self._uaux, 0)
357              qdot_zero = dict.fromkeys(qdot, 0)
358  
359              # Extract the linear coefficient matrices as per the following
360              # equation:
361              #
362              # k_ku(q,t)*u(t) + k_kqdot(q,t)*q'(t) + f_k(q,t) = 0
363              #
364              k_ku = kdeqs.jacobian(u)
365              k_kqdot = kdeqs.jacobian(qdot)
366              f_k = kdeqs.xreplace(u_zero).xreplace(qdot_zero)
367  
368              # The kinematic differential equations should be linear in both q'
369              # and u so check for u and q' in the components.
370              dy_syms = find_dynamicsymbols(k_ku.row_join(k_kqdot).row_join(f_k))
371              nonlin_vars = [vari for vari in u[:] + qdot[:] if vari in dy_syms]
372              if nonlin_vars:
373                  msg = ('The provided kinematic differential equations are '
374                         'nonlinear in {}. They must be linear in the '
375                         'generalized speeds and derivatives of the generalized '
376                         'coordinates.')
377                  raise ValueError(msg.format(nonlin_vars))
378  
379              self._f_k_implicit = f_k.xreplace(uaux_zero)
380              self._k_ku_implicit = k_ku.xreplace(uaux_zero)
381              self._k_kqdot_implicit = k_kqdot
382  
383              # Solve for q'(t) such that the coefficient matrices are now in
384              # this form:
385              #
386              # k_kqdot^-1*k_ku*u(t) + I*q'(t) + k_kqdot^-1*f_k = 0
387              #
388              # NOTE : Solving the kinematic differential equations here is not
389              # necessary and prevents the equations from being provided in fully
390              # implicit form.
391              f_k_explicit = linear_solver(k_kqdot, f_k)
392              k_ku_explicit = linear_solver(k_kqdot, k_ku)
393              self._qdot_u_map = dict(zip(qdot, -(k_ku_explicit*u + f_k_explicit)))
394  
395              self._f_k = f_k_explicit.xreplace(uaux_zero)
396              self._k_ku = k_ku_explicit.xreplace(uaux_zero)
397              self._k_kqdot = eye(len(qdot))
398  
399          else:
400              self._qdot_u_map = None
401              self._f_k_implicit = self._f_k = Matrix()
402              self._k_ku_implicit = self._k_ku = Matrix()
403              self._k_kqdot_implicit = self._k_kqdot = Matrix()
404  
405      def _form_fr(self, fl):
406          """Form the generalized active force."""
407          if fl is not None and (len(fl) == 0 or not iterable(fl)):
408              raise ValueError('Force pairs must be supplied in an '
409                  'non-empty iterable or None.')
410  
411          N = self._inertial
412          # pull out relevant velocities for constructing partial velocities
413          vel_list, f_list = _f_list_parser(fl, N)
414          vel_list = [msubs(i, self._qdot_u_map) for i in vel_list]
415          f_list = [msubs(i, self._qdot_u_map) for i in f_list]
416  
417          # Fill Fr with dot product of partial velocities and forces
418          o = len(self.u)
419          b = len(f_list)
420          FR = zeros(o, 1)
421          partials = partial_velocity(vel_list, self.u, N)
422          for i in range(o):
423              FR[i] = sum(partials[j][i].dot(f_list[j]) for j in range(b))
424  
425          # In case there are dependent speeds
426          if self._udep:
427              p = o - len(self._udep)
428              FRtilde = FR[:p, 0]
429              FRold = FR[p:o, 0]
430              FRtilde += self._Ars.T * FRold
431              FR = FRtilde
432  
433          self._forcelist = fl
434          self._fr = FR
435          return FR

437      def _form_frstar(self, bl):
438          """Form the generalized inertia force."""
439  
440          if not iterable(bl):
441              raise TypeError('Bodies must be supplied in an iterable.')
442  
443          t = dynamicsymbols._t
444          N = self._inertial
445          # Dicts setting things to zero
446          udot_zero = dict.fromkeys(self._udot, 0)
447          uaux_zero = dict.fromkeys(self._uaux, 0)
448          uauxdot = [diff(i, t) for i in self._uaux]
449          uauxdot_zero = dict.fromkeys(uauxdot, 0)
450          # Dictionary of q' and q'' to u and u'
451          q_ddot_u_map = {k.diff(t): v.diff(t).xreplace(
452              self._qdot_u_map) for (k, v) in self._qdot_u_map.items()}
453          q_ddot_u_map.update(self._qdot_u_map)
454  
455          # Fill up the list of partials: format is a list with num elements
456          # equal to number of entries in body list. Each of these elements is a
457          # list - either of length 1 for the translational components of
458          # particles or of length 2 for the translational and rotational
459          # components of rigid bodies. The inner most list is the list of
460          # partial velocities.
461          def get_partial_velocity(body):
462              if isinstance(body, RigidBody):
463                  vlist = [body.masscenter.vel(N), body.frame.ang_vel_in(N)]
464              elif isinstance(body, Particle):
465                  vlist = [body.point.vel(N),]
466              else:
467                  raise TypeError('The body list may only contain either '
468                                  'RigidBody or Particle as list elements.')
469              v = [msubs(vel, self._qdot_u_map) for vel in vlist]
470              return partial_velocity(v, self.u, N)
471          partials = [get_partial_velocity(body) for body in bl]
472  
473          # Compute fr_star in two components:
474          # fr_star = -(MM*u' + nonMM)
475          o = len(self.u)
476          MM = zeros(o, o)
477          nonMM = zeros(o, 1)
478          zero_uaux = lambda expr: msubs(expr, uaux_zero)
479          zero_udot_uaux = lambda expr: msubs(msubs(expr, udot_zero), uaux_zero)
480          for i, body in enumerate(bl):
481              if isinstance(body, RigidBody):
482                  M = zero_uaux(body.mass)
483                  I = zero_uaux(body.central_inertia)
484                  vel = zero_uaux(body.masscenter.vel(N))
485                  omega = zero_uaux(body.frame.ang_vel_in(N))
486                  acc = zero_udot_uaux(body.masscenter.acc(N))
487                  inertial_force = (M.diff(t) * vel + M * acc)
488                  inertial_torque = zero_uaux((I.dt(body.frame).dot(omega)) +
489                      msubs(I.dot(body.frame.ang_acc_in(N)), udot_zero) +
490                      (omega.cross(I.dot(omega))))
491                  for j in range(o):
492                      tmp_vel = zero_uaux(partials[i][0][j])
493                      tmp_ang = zero_uaux(I.dot(partials[i][1][j]))
494                      for k in range(o):
495                          # translational
496                          MM[j, k] += M*tmp_vel.dot(partials[i][0][k])
497                          # rotational
498                          MM[j, k] += tmp_ang.dot(partials[i][1][k])
499                      nonMM[j] += inertial_force.dot(partials[i][0][j])
500                      nonMM[j] += inertial_torque.dot(partials[i][1][j])
501              else:
502                  M = zero_uaux(body.mass)
503                  vel = zero_uaux(body.point.vel(N))
504                  acc = zero_udot_uaux(body.point.acc(N))
505                  inertial_force = (M.diff(t) * vel + M * acc)
506                  for j in range(o):
507                      temp = zero_uaux(partials[i][0][j])
508                      for k in range(o):
509                          MM[j, k] += M*temp.dot(partials[i][0][k])
510                      nonMM[j] += inertial_force.dot(partials[i][0][j])
511          # Compose fr_star out of MM and nonMM
512          MM = zero_uaux(msubs(MM, q_ddot_u_map))
513          nonMM = msubs(msubs(nonMM, q_ddot_u_map),
514                  udot_zero, uauxdot_zero, uaux_zero)
515          fr_star = -(MM * msubs(Matrix(self._udot), uauxdot_zero) + nonMM)
516  
517          # If there are dependent speeds, we need to find fr_star_tilde
518          if self._udep:
519              p = o - len(self._udep)
520              fr_star_ind = fr_star[:p, 0]
521              fr_star_dep = fr_star[p:o, 0]
522              fr_star = fr_star_ind + (self._Ars.T * fr_star_dep)
523              # Apply the same to MM
524              MMi = MM[:p, :]
525              MMd = MM[p:o, :]
526              MM = MMi + (self._Ars.T * MMd)
527              # Apply the same to nonMM
528              nonMM = nonMM[:p, :] + (self._Ars.T * nonMM[p:o, :])
529  
530          self._bodylist = bl
531          self._frstar = fr_star
532          self._k_d = MM
533          self._f_d = -(self._fr - nonMM)
534          return fr_star

536      def to_linearizer(self, linear_solver='LU'):
537          """Returns an instance of the Linearizer class, initiated from the 
538          data in the KanesMethod class. This may be more desirable than using 
539          the linearize class method, as the Linearizer object will allow more 
540          efficient recalculation (i.e. about varying operating points). 
541   
542          Parameters 
543          ========== 
544          linear_solver : str, callable 
545              Method used to solve the several symbolic linear systems of the 
546              form ``A*x=b`` in the linearization process. If a string is 
547              supplied, it should be a valid method that can be used with the 
548              :meth:`sympy.matrices.matrixbase.MatrixBase.solve`. If a callable is 
549              supplied, it should have the format ``x = f(A, b)``, where it 
550              solves the equations and returns the solution. The default is 
551              ``'LU'`` which corresponds to SymPy's ``A.LUsolve(b)``. 
552              ``LUsolve()`` is fast to compute but will often result in 
553              divide-by-zero and thus ``nan`` results. 
554   
555          Returns 
556          ======= 
557          Linearizer 
558              An instantiated 
559              :class:`sympy.physics.mechanics.linearize.Linearizer`. 
560   
561          """
562  
563          if (self._fr is None) or (self._frstar is None):
564              raise ValueError('Need to compute Fr, Fr* first.')
565  
566          # Get required equation components. The Kane's method class breaks
567          # these into pieces. Need to reassemble
568          f_c = self._f_h
569          if self._f_nh and self._k_nh:
570              f_v = self._f_nh + self._k_nh*Matrix(self.u)
571          else:
572              f_v = Matrix()
573          if self._f_dnh and self._k_dnh:
574              f_a = self._f_dnh + self._k_dnh*Matrix(self._udot)
575          else:
576              f_a = Matrix()
577          # Dicts to sub to zero, for splitting up expressions
578          u_zero = dict.fromkeys(self.u, 0)
579          ud_zero = dict.fromkeys(self._udot, 0)
580          qd_zero = dict.fromkeys(self._qdot, 0)
581          qd_u_zero = dict.fromkeys(Matrix([self._qdot, self.u]), 0)
582          # Break the kinematic differential eqs apart into f_0 and f_1
583          f_0 = msubs(self._f_k, u_zero) + self._k_kqdot*Matrix(self._qdot)
584          f_1 = msubs(self._f_k, qd_zero) + self._k_ku*Matrix(self.u)
585          # Break the dynamic differential eqs into f_2 and f_3
586          f_2 = msubs(self._frstar, qd_u_zero)
587          f_3 = msubs(self._frstar, ud_zero) + self._fr
588          f_4 = zeros(len(f_2), 1)
589  
590          # Get the required vector components
591          q = self.q
592          u = self.u
593          if self._qdep:
594              q_i = q[:-len(self._qdep)]
595          else:
596              q_i = q
597          q_d = self._qdep
598          if self._udep:
599              u_i = u[:-len(self._udep)]
600          else:
601              u_i = u
602          u_d = self._udep
603  
604          # Form dictionary to set auxiliary speeds & their derivatives to 0.
605          uaux = self._uaux
606          uauxdot = uaux.diff(dynamicsymbols._t)
607          uaux_zero = dict.fromkeys(Matrix([uaux, uauxdot]), 0)
608  
609          # Checking for dynamic symbols outside the dynamic differential
610          # equations; throws error if there is.
611          sym_list = set(Matrix([q, self._qdot, u, self._udot, uaux, uauxdot]))
612          if any(find_dynamicsymbols(i, sym_list) for i in [self._k_kqdot,
613                  self._k_ku, self._f_k, self._k_dnh, self._f_dnh, self._k_d]):
614              raise ValueError('Cannot have dynamicsymbols outside dynamic \
615                               forcing vector.')
616  
617          # Find all other dynamic symbols, forming the forcing vector r.
618          # Sort r to make it canonical.
619          r = list(find_dynamicsymbols(msubs(self._f_d, uaux_zero), sym_list))
620          r.sort(key=default_sort_key)
621  
622          # Check for any derivatives of variables in r that are also found in r.
623          for i in r:
624              if diff(i, dynamicsymbols._t) in r:
625                  raise ValueError('Cannot have derivatives of specified \
626                                   quantities when linearizing forcing terms.')
627          return Linearizer(f_0, f_1, f_2, f_3, f_4, f_c, f_v, f_a, q, u, q_i,
628                  q_d, u_i, u_d, r, linear_solver=linear_solver)
629

631      def linearize(self, *, new_method=None, linear_solver='LU', **kwargs):
632          """ Linearize the equations of motion about a symbolic operating point. 
633   
634          Parameters 
635          ========== 
636          new_method 
637              Deprecated, does nothing and will be removed. 
638          linear_solver : str, callable 
639              Method used to solve the several symbolic linear systems of the 
640              form ``A*x=b`` in the linearization process. If a string is 
641              supplied, it should be a valid method that can be used with the 
642              :meth:`sympy.matrices.matrixbase.MatrixBase.solve`. If a callable is 
643              supplied, it should have the format ``x = f(A, b)``, where it 
644              solves the equations and returns the solution. The default is 
645              ``'LU'`` which corresponds to SymPy's ``A.LUsolve(b)``. 
646              ``LUsolve()`` is fast to compute but will often result in 
647              divide-by-zero and thus ``nan`` results. 
648          **kwargs 
649              Extra keyword arguments are passed to 
650              :meth:`sympy.physics.mechanics.linearize.Linearizer.linearize`. 
651   
652          Explanation 
653          =========== 
654   
655          If kwarg A_and_B is False (default), returns M, A, B, r for the 
656          linearized form, M*[q', u']^T = A*[q_ind, u_ind]^T + B*r. 
657   
658          If kwarg A_and_B is True, returns A, B, r for the linearized form 
659          dx = A*x + B*r, where x = [q_ind, u_ind]^T. Note that this is 
660          computationally intensive if there are many symbolic parameters. For 
661          this reason, it may be more desirable to use the default A_and_B=False, 
662          returning M, A, and B. Values may then be substituted in to these 
663          matrices, and the state space form found as 
664          A = P.T*M.inv()*A, B = P.T*M.inv()*B, where P = Linearizer.perm_mat. 
665   
666          In both cases, r is found as all dynamicsymbols in the equations of 
667          motion that are not part of q, u, q', or u'. They are sorted in 
668          canonical form. 
669   
670          The operating points may be also entered using the ``op_point`` kwarg. 
671          This takes a dictionary of {symbol: value}, or a an iterable of such 
672          dictionaries. The values may be numeric or symbolic. The more values 
673          you can specify beforehand, the faster this computation will run. 
674   
675          For more documentation, please see the ``Linearizer`` class. 
676   
677          """
678  
679          linearizer = self.to_linearizer(linear_solver=linear_solver)
680          result = linearizer.linearize(**kwargs)
681          return result + (linearizer.r,)

683      def kanes_equations(self, bodies=None, loads=None):
684          """ Method to form Kane's equations, Fr + Fr* = 0. 
685   
686          Explanation 
687          =========== 
688   
689          Returns (Fr, Fr*). In the case where auxiliary generalized speeds are 
690          present (say, s auxiliary speeds, o generalized speeds, and m motion 
691          constraints) the length of the returned vectors will be o - m + s in 
692          length. The first o - m equations will be the constrained Kane's 
693          equations, then the s auxiliary Kane's equations. These auxiliary 
694          equations can be accessed with the auxiliary_eqs property. 
695   
696          Parameters 
697          ========== 
698   
699          bodies : iterable 
700              An iterable of all RigidBody's and Particle's in the system. 
701              A system must have at least one body. 
702          loads : iterable 
703              Takes in an iterable of (Particle, Vector) or (ReferenceFrame, Vector) 
704              tuples which represent the force at a point or torque on a frame. 
705              Must be either a non-empty iterable of tuples or None which corresponds 
706              to a system with no constraints. 
707          """
708          if bodies is None:
709              bodies = self.bodies
710          if  loads is None and self._forcelist is not None:
711              loads = self._forcelist
712          if loads == []:
713              loads = None
714          if not self._k_kqdot:
715              raise AttributeError('Create an instance of KanesMethod with '
716                      'kinematic differential equations to use this method.')
717          fr = self._form_fr(loads)
718          frstar = self._form_frstar(bodies)
719          if self._uaux:
720              if not self._udep:
721                  km = KanesMethod(self._inertial, self.q, self._uaux,
722                               u_auxiliary=self._uaux, constraint_solver=self._constraint_solver)
723              else:
724                  km = KanesMethod(self._inertial, self.q, self._uaux,
725                          u_auxiliary=self._uaux, u_dependent=self._udep,
726                          velocity_constraints=(self._k_nh * self.u +
727                          self._f_nh),
728                          acceleration_constraints=(self._k_dnh * self._udot +
729                          self._f_dnh),
730                          constraint_solver=self._constraint_solver
731                          )
732              km._qdot_u_map = self._qdot_u_map
733              self._km = km
734              fraux = km._form_fr(loads)
735              frstaraux = km._form_frstar(bodies)
736              self._aux_eq = fraux + frstaraux
737              self._fr = fr.col_join(fraux)
738              self._frstar = frstar.col_join(frstaraux)
739          return (self._fr, self._frstar)
740  
741      def _form_eoms(self):
742          fr, frstar = self.kanes_equations(self.bodylist, self.forcelist)
743          return fr + frstar

745      def rhs(self, inv_method=None):
746          """Returns the system's equations of motion in first order form. The 
747          output is the right hand side of:: 
748   
749             x' = |q'| =: f(q, u, r, p, t) 
750                  |u'| 
751   
752          The right hand side is what is needed by most numerical ODE 
753          integrators. 
754   
755          Parameters 
756          ========== 
757   
758          inv_method : str 
759              The specific sympy inverse matrix calculation method to use. For a 
760              list of valid methods, see 
761              :meth:`~sympy.matrices.matrixbase.MatrixBase.inv` 
762   
763          """
764          rhs = zeros(len(self.q) + len(self.u), 1)
765          kdes = self.kindiffdict()
766          for i, q_i in enumerate(self.q):
767              rhs[i] = kdes[q_i.diff()]
768  
769          if inv_method is None:
770              rhs[len(self.q):, 0] = self.mass_matrix.LUsolve(self.forcing)
771          else:
772              rhs[len(self.q):, 0] = (self.mass_matrix.inv(inv_method,
773                                                           try_block_diag=True) *
774                                      self.forcing)
775  
776          return rhs
777  
778      def kindiffdict(self):
779          """Returns a dictionary mapping q' to u."""
780          if not self._qdot_u_map:
781              raise AttributeError('Create an instance of KanesMethod with '
782                      'kinematic differential equations to use this method.')
783          return self._qdot_u_map

795      def mass_matrix_kin(self):
796          r"""The kinematic "mass matrix" $\mathbf{k_{k\dot{q}}}$ of the system."""
797          return self._k_kqdot if self.explicit_kinematics else self._k_kqdot_implicit

799      @property
800      def forcing_kin(self):
801          """The kinematic "forcing vector" of the system."""
802          if self.explicit_kinematics:
803              return -(self._k_ku * Matrix(self.u) + self._f_k)
804          else:
805              return -(self._k_ku_implicit * Matrix(self.u) + self._f_k_implicit)

808      def mass_matrix(self):
809          """The mass matrix of the system."""
810          if not self._fr or not self._frstar:
811              raise ValueError('Need to compute Fr, Fr* first.')
812          return Matrix([self._k_d, self._k_dnh])

815      def forcing(self):
816          """The forcing vector of the system."""
817          if not self._fr or not self._frstar:
818              raise ValueError('Need to compute Fr, Fr* first.')
819          return -Matrix([self._f_d, self._f_dnh])

821      @property
822      def mass_matrix_full(self):
823          """The mass matrix of the system, augmented by the kinematic 
824          differential equations in explicit or implicit form."""

826              raise ValueError('Need to compute Fr, Fr* first.')

829              zeros(o, n).row_join(self.mass_matrix))
