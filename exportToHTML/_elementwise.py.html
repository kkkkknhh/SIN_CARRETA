6    def find_root(f, init, /, *, args=(), tolerances=None, maxiter=None, callback=None):
7        """Find the root of a monotonic, real-valued function of a real variable. 
8     
9        For each element of the output of `f`, `find_root` seeks the scalar 
10       root that makes the element 0. This function currently uses Chandrupatla's 
11       bracketing algorithm [1]_ and therefore requires argument `init` to 
12       provide a bracket around the root: the function values at the two endpoints 
13       must have opposite signs. 
14    
15       Provided a valid bracket, `find_root` is guaranteed to converge to a solution 
16       that satisfies the provided `tolerances` if the function is continuous within 
17       the bracket. 
18    
19       This function works elementwise when `init` and `args` contain (broadcastable) 
20       arrays. 
21    
22       Parameters 
23       ---------- 
24       f : callable 
25           The function whose root is desired. The signature must be:: 
26    
27               f(x: array, *args) -> array 
28    
29           where each element of ``x`` is a finite real and ``args`` is a tuple, 
30           which may contain an arbitrary number of arrays that are broadcastable 
31           with ``x``. 
32    
33           `f` must be an elementwise function: each element ``f(x)[i]`` 
34           must equal ``f(x[i])`` for all indices ``i``. It must not mutate the 
35           array ``x`` or the arrays in ``args``. 
36    
37           `find_root` seeks an array ``x`` such that ``f(x)`` is an array of zeros. 
38       init : 2-tuple of float array_like 
39           The lower and upper endpoints of a bracket surrounding the desired root. 
40           A bracket is valid if arrays ``xl, xr = init`` satisfy ``xl < xr`` and 
41           ``sign(f(xl)) == -sign(f(xr))`` elementwise. Arrays be broadcastable with 
42           one another and `args`. 
43       args : tuple of array_like, optional 
44           Additional positional array arguments to be passed to `f`. Arrays 
45           must be broadcastable with one another and the arrays of `init`. 
46           If the callable for which the root is desired requires arguments that are 
47           not broadcastable with `x`, wrap that callable with `f` such that `f` 
48           accepts only `x` and broadcastable ``*args``. 
49       tolerances : dictionary of floats, optional 
50           Absolute and relative tolerances on the root and function value. 
51           Valid keys of the dictionary are: 
52    
53           - ``xatol`` - absolute tolerance on the root 
54           - ``xrtol`` - relative tolerance on the root 
55           - ``fatol`` - absolute tolerance on the function value 
56           - ``frtol`` - relative tolerance on the function value 
57    
58           See Notes for default values and explicit termination conditions. 
59       maxiter : int, optional 
60           The maximum number of iterations of the algorithm to perform. 
61           The default is the maximum possible number of bisections within 
62           the (normal) floating point numbers of the relevant dtype. 
63       callback : callable, optional 
64           An optional user-supplied function to be called before the first 
65           iteration and after each iteration. 
66           Called as ``callback(res)``, where ``res`` is a ``_RichResult`` 
67           similar to that returned by `find_root` (but containing the current 
68           iterate's values of all variables). If `callback` raises a 
69           ``StopIteration``, the algorithm will terminate immediately and 
70           `find_root` will return a result. `callback` must not mutate 
71           `res` or its attributes. 
72    
73       Returns 
74       ------- 
75       res : _RichResult 
76           An object similar to an instance of `scipy.optimize.OptimizeResult` with the 
77           following attributes. The descriptions are written as though the values will 
78           be scalars; however, if `f` returns an array, the outputs will be 
79           arrays of the same shape. 
80    
81           success : bool array 
82               ``True`` where the algorithm terminated successfully (status ``0``); 
83               ``False`` otherwise. 
84           status : int array 
85               An integer representing the exit status of the algorithm. 
86    
87               - ``0`` : The algorithm converged to the specified tolerances. 
88               - ``-1`` : The initial bracket was invalid. 
89               - ``-2`` : The maximum number of iterations was reached. 
90               - ``-3`` : A non-finite value was encountered. 
91               - ``-4`` : Iteration was terminated by `callback`. 
92               - ``1`` : The algorithm is proceeding normally (in `callback` only). 
93    
94           x : float array 
95               The root of the function, if the algorithm terminated successfully. 
96           f_x : float array 
97               The value of `f` evaluated at `x`. 
98           nfev : int array 
99               The number of abscissae at which `f` was evaluated to find the root. 
100              This is distinct from the number of times `f` is *called* because the 
101              the function may evaluated at multiple points in a single call. 
102          nit : int array 
103              The number of iterations of the algorithm that were performed. 
104          bracket : tuple of float arrays 
105              The lower and upper endpoints of the final bracket. 
106          f_bracket : tuple of float arrays 
107              The value of `f` evaluated at the lower and upper endpoints of the 
108              bracket. 
109   
110      Notes 
111      ----- 
112      Implemented based on Chandrupatla's original paper [1]_. 
113   
114      Let: 
115   
116      -  ``a, b = init`` be the left and right endpoints of the initial bracket, 
117      - ``xl`` and ``xr`` be the left and right endpoints of the final bracket, 
118      - ``xmin = xl if abs(f(xl)) <= abs(f(xr)) else xr`` be the final bracket 
119        endpoint with the smaller function value, and 
120      - ``fmin0 = min(f(a), f(b))`` be the minimum of the two values of the 
121        function evaluated at the initial bracket endpoints. 
122   
123      Then the algorithm is considered to have converged when 
124   
125      - ``abs(xr - xl) < xatol + abs(xmin) * xrtol`` or 
126      - ``fun(xmin) <= fatol + abs(fmin0) * frtol``. 
127   
128      This is equivalent to the termination condition described in [1]_ with 
129      ``xrtol = 4e-10``, ``xatol = 1e-5``, and ``fatol = frtol = 0``. 
130      However, the default values of the `tolerances` dictionary are 
131      ``xatol = 4*tiny``, ``xrtol = 4*eps``, ``frtol = 0``, and ``fatol = tiny``, 
132      where ``eps`` and ``tiny`` are the precision and smallest normal number 
133      of the result ``dtype`` of function inputs and outputs. 
134   
135      References 
136      ---------- 
137   
138      .. [1] Chandrupatla, Tirupathi R. 
139          "A new hybrid quadratic/bisection algorithm for finding the zero of a 
140          nonlinear function without using derivatives". 
141          Advances in Engineering Software, 28(3), 145-149. 
142          https://doi.org/10.1016/s0965-9978(96)00051-8 
143   
144      See Also 
145      -------- 
146      bracket_root 
147   
148      Examples 
149      -------- 
150      Suppose we wish to find the root of the following function. 
151   
152      >>> def f(x, c=5): 
153      ...     return x**3 - 2*x - c 
154   
155      First, we must find a valid bracket. The function is not monotonic, 
156      but `bracket_root` may be able to provide a bracket. 
157   
158      >>> from scipy.optimize import elementwise 
159      >>> res_bracket = elementwise.bracket_root(f, 0) 
160      >>> res_bracket.success 
161      True 
162      >>> res_bracket.bracket 
163      (2.0, 4.0) 
164   
165      Indeed, the values of the function at the bracket endpoints have 
166      opposite signs. 
167   
168      >>> res_bracket.f_bracket 
169      (-1.0, 51.0) 
170   
171      Once we have a valid bracket, `find_root` can be used to provide 
172      a precise root. 
173   
174      >>> res_root = elementwise.find_root(f, res_bracket.bracket) 
175      >>> res_root.x 
176      2.0945514815423265 
177   
178      The final bracket is only a few ULPs wide, so the error between 
179      this value and the true root cannot be much smaller within values 
180      that are representable in double precision arithmetic. 
181   
182      >>> import numpy as np 
183      >>> xl, xr = res_root.bracket 
184      >>> (xr - xl) / np.spacing(xl) 
185      2.0 
186      >>> res_root.f_bracket 
187      (-8.881784197001252e-16, 9.769962616701378e-15) 
188   
189      `bracket_root` and `find_root` accept arrays for most arguments. 
190      For instance, to find the root for a few values of the parameter ``c`` 
191      at once: 
192   
193      >>> c = np.asarray([3, 4, 5]) 
194      >>> res_bracket = elementwise.bracket_root(f, 0, args=(c,)) 
195      >>> res_bracket.bracket 
196      (array([1., 1., 2.]), array([2., 2., 4.])) 
197      >>> res_root = elementwise.find_root(f, res_bracket.bracket, args=(c,)) 
198      >>> res_root.x 
199      array([1.8932892 , 2.        , 2.09455148]) 
200   
201      """
202  
203      def reformat_result(res_in):
204          res_out = _RichResult()
205          res_out.status = res_in.status
206          res_out.success = res_in.success
207          res_out.x = res_in.x
208          res_out.f_x = res_in.fun
209          res_out.nfev = res_in.nfev
210          res_out.nit = res_in.nit
211          res_out.bracket = (res_in.xl, res_in.xr)
212          res_out.f_bracket = (res_in.fl, res_in.fr)
213          res_out._order_keys = ['success', 'status', 'x', 'f_x',
214                                 'nfev', 'nit', 'bracket', 'f_bracket']
215          return res_out
216  
217      xl, xr = init
218      default_tolerances = dict(xatol=None, xrtol=None, fatol=None, frtol=0)
219      tolerances = {} if tolerances is None else tolerances
220      default_tolerances.update(tolerances)
221      tolerances = default_tolerances
222  
223      if callable(callback):
224          def _callback(res):
225              return callback(reformat_result(res))
226      else:
227          _callback = callback
228  
229      res = _chandrupatla(f, xl, xr, args=args, **tolerances,
230                          maxiter=maxiter, callback=_callback)
231      return reformat_result(res)

234  def find_minimum(f, init, /, *, args=(), tolerances=None, maxiter=100, callback=None):
235      """Find the minimum of an unimodal, real-valued function of a real variable. 
236   
237      For each element of the output of `f`, `find_minimum` seeks the scalar minimizer 
238      that minimizes the element. This function currently uses Chandrupatla's 
239      bracketing minimization algorithm [1]_ and therefore requires argument `init` 
240      to provide a three-point minimization bracket: ``x1 < x2 < x3`` such that 
241      ``func(x1) >= func(x2) <= func(x3)``, where one of the inequalities is strict. 
242   
243      Provided a valid bracket, `find_minimum` is guaranteed to converge to a local 
244      minimum that satisfies the provided `tolerances` if the function is continuous 
245      within the bracket. 
246   
247      This function works elementwise when `init` and `args` contain (broadcastable) 
248      arrays. 
249   
250      Parameters 
251      ---------- 
252      f : callable 
253          The function whose minimizer is desired. The signature must be:: 
254   
255              f(x: array, *args) -> array 
256   
257          where each element of ``x`` is a finite real and ``args`` is a tuple, 
258          which may contain an arbitrary number of arrays that are broadcastable 
259          with ``x``. 
260   
261          `f` must be an elementwise function: each element ``f(x)[i]`` 
262          must equal ``f(x[i])`` for all indices ``i``. It must not mutate the 
263          array ``x`` or the arrays in ``args``. 
264   
265          `find_minimum` seeks an array ``x`` such that ``f(x)`` is an array of 
266          local minima. 
267      init : 3-tuple of float array_like 
268          The abscissae of a standard scalar minimization bracket. A bracket is 
269          valid if arrays ``x1, x2, x3 = init`` satisfy ``x1 < x2 < x3`` and 
270          ``func(x1) >= func(x2) <= func(x3)``, where one of the inequalities 
271          is strict. Arrays must be broadcastable with one another and the arrays 
272          of `args`. 
273      args : tuple of array_like, optional 
274          Additional positional array arguments to be passed to `f`. Arrays 
275          must be broadcastable with one another and the arrays of `init`. 
276          If the callable for which the root is desired requires arguments that are 
277          not broadcastable with `x`, wrap that callable with `f` such that `f` 
278          accepts only `x` and broadcastable ``*args``. 
279      tolerances : dictionary of floats, optional 
280          Absolute and relative tolerances on the root and function value. 
281          Valid keys of the dictionary are: 
282   
283          - ``xatol`` - absolute tolerance on the root 
284          - ``xrtol`` - relative tolerance on the root 
285          - ``fatol`` - absolute tolerance on the function value 
286          - ``frtol`` - relative tolerance on the function value 
287   
288          See Notes for default values and explicit termination conditions. 
289      maxiter : int, default: 100 
290          The maximum number of iterations of the algorithm to perform. 
291      callback : callable, optional 
292          An optional user-supplied function to be called before the first 
293          iteration and after each iteration. 
294          Called as ``callback(res)``, where ``res`` is a ``_RichResult`` 
295          similar to that returned by `find_minimum` (but containing the current 
296          iterate's values of all variables). If `callback` raises a 
297          ``StopIteration``, the algorithm will terminate immediately and 
298          `find_root` will return a result. `callback` must not mutate 
299          `res` or its attributes. 
300   
301      Returns 
302      ------- 
303      res : _RichResult 
304          An object similar to an instance of `scipy.optimize.OptimizeResult` with the 
305          following attributes. The descriptions are written as though the values will 
306          be scalars; however, if `f` returns an array, the outputs will be 
307          arrays of the same shape. 
308   
309          success : bool array 
310              ``True`` where the algorithm terminated successfully (status ``0``); 
311              ``False`` otherwise. 
312          status : int array 
313              An integer representing the exit status of the algorithm. 
314   
315              - ``0`` : The algorithm converged to the specified tolerances. 
316              - ``-1`` : The algorithm encountered an invalid bracket. 
317              - ``-2`` : The maximum number of iterations was reached. 
318              - ``-3`` : A non-finite value was encountered. 
319              - ``-4`` : Iteration was terminated by `callback`. 
320              - ``1`` : The algorithm is proceeding normally (in `callback` only). 
321   
322          x : float array 
323              The minimizer of the function, if the algorithm terminated successfully. 
324          f_x : float array 
325              The value of `f` evaluated at `x`. 
326          nfev : int array 
327              The number of abscissae at which `f` was evaluated to find the root. 
328              This is distinct from the number of times `f` is *called* because the 
329              the function may evaluated at multiple points in a single call. 
330          nit : int array 
331              The number of iterations of the algorithm that were performed. 
332          bracket : tuple of float arrays 
333              The final three-point bracket. 
334          f_bracket : tuple of float arrays 
335              The value of `f` evaluated at the bracket points. 
336   
337      Notes 
338      ----- 
339      Implemented based on Chandrupatla's original paper [1]_. 
340   
341      If ``xl < xm < xr`` are the points of the bracket and ``fl >= fm <= fr`` 
342      (where one of the inequalities is strict) are the values of `f` evaluated 
343      at those points, then the algorithm is considered to have converged when: 
344   
345      - ``xr - xl <= abs(xm)*xrtol + xatol`` or 
346      - ``(fl - 2*fm + fr)/2 <= abs(fm)*frtol + fatol``. 
347   
348      Note that first of these differs from the termination conditions described 
349      in [1]_. 
350   
351      The default value of `xrtol` is the square root of the precision of the 
352      appropriate dtype, and ``xatol = fatol = frtol`` is the smallest normal 
353      number of the appropriate dtype. 
354   
355      References 
356      ---------- 
357   
358      .. [1] Chandrupatla, Tirupathi R. (1998). 
359          "An efficient quadratic fit-sectioning algorithm for minimization 
360          without derivatives". 
361          Computer Methods in Applied Mechanics and Engineering, 152 (1-2), 
362          211-217. https://doi.org/10.1016/S0045-7825(97)00190-4 
363   
364      See Also 
365      -------- 
366      bracket_minimum 
367   
368      Examples 
369      -------- 
370      Suppose we wish to minimize the following function. 
371   
372      >>> def f(x, c=1): 
373      ...     return (x - c)**2 + 2 
374   
375      First, we must find a valid bracket. The function is unimodal, 
376      so `bracket_minium` will easily find a bracket. 
377   
378      >>> from scipy.optimize import elementwise 
379      >>> res_bracket = elementwise.bracket_minimum(f, 0) 
380      >>> res_bracket.success 
381      True 
382      >>> res_bracket.bracket 
383      (0.0, 0.5, 1.5) 
384   
385      Indeed, the bracket points are ordered and the function value 
386      at the middle bracket point is less than at the surrounding 
387      points. 
388   
389      >>> xl, xm, xr = res_bracket.bracket 
390      >>> fl, fm, fr = res_bracket.f_bracket 
391      >>> (xl < xm < xr) and (fl > fm <= fr) 
392      True 
393   
394      Once we have a valid bracket, `find_minimum` can be used to provide 
395      an estimate of the minimizer. 
396   
397      >>> res_minimum = elementwise.find_minimum(f, res_bracket.bracket) 
398      >>> res_minimum.x 
399      1.0000000149011612 
400   
401      The function value changes by only a few ULPs within the bracket, so 
402      the minimizer cannot be determined much more precisely by evaluating 
403      the function alone (i.e. we would need its derivative to do better). 
404   
405      >>> import numpy as np 
406      >>> fl, fm, fr = res_minimum.f_bracket 
407      >>> (fl - fm) / np.spacing(fm), (fr - fm) / np.spacing(fm) 
408      (0.0, 2.0) 
409   
410      Therefore, a precise minimum of the function is given by: 
411   
412      >>> res_minimum.f_x 
413      2.0 
414   
415      `bracket_minimum` and `find_minimum` accept arrays for most arguments. 
416      For instance, to find the minimizers and minima for a few values of the 
417      parameter ``c`` at once: 
418   
419      >>> c = np.asarray([1, 1.5, 2]) 
420      >>> res_bracket = elementwise.bracket_minimum(f, 0, args=(c,)) 
421      >>> res_bracket.bracket 
422      (array([0. , 0.5, 0.5]), array([0.5, 1.5, 1.5]), array([1.5, 2.5, 2.5])) 
423      >>> res_minimum = elementwise.find_minimum(f, res_bracket.bracket, args=(c,)) 
424      >>> res_minimum.x 
425      array([1.00000001, 1.5       , 2.        ]) 
426      >>> res_minimum.f_x 
427      array([2., 2., 2.]) 
428   
429      """
430  
431      def reformat_result(res_in):
432          res_out = _RichResult()
433          res_out.status = res_in.status
434          res_out.success = res_in.success
435          res_out.x = res_in.x
436          res_out.f_x = res_in.fun
437          res_out.nfev = res_in.nfev
438          res_out.nit = res_in.nit
439          res_out.bracket = (res_in.xl, res_in.xm, res_in.xr)
440          res_out.f_bracket = (res_in.fl, res_in.fm, res_in.fr)
441          res_out._order_keys = ['success', 'status', 'x', 'f_x',
442                                 'nfev', 'nit', 'bracket', 'f_bracket']
443          return res_out
444  
445      xl, xm, xr = init
446      default_tolerances = dict(xatol=None, xrtol=None, fatol=None, frtol=None)
447      tolerances = {} if tolerances is None else tolerances
448      default_tolerances.update(tolerances)
449      tolerances = default_tolerances
450  
451      if callable(callback):
452          def _callback(res):
453              return callback(reformat_result(res))
454      else:
455          _callback = callback
456  
457      res = _chandrupatla_minimize(f, xl, xm, xr, args=args, **tolerances,
458                                   maxiter=maxiter, callback=_callback)
459      return reformat_result(res)

462  def bracket_root(f, xl0, xr0=None, *, xmin=None, xmax=None, factor=None, args=(),
463                   maxiter=1000):
464      """Bracket the root of a monotonic, real-valued function of a real variable. 
465   
466      For each element of the output of `f`, `bracket_root` seeks the scalar 
467      bracket endpoints ``xl`` and ``xr`` such that ``sign(f(xl)) == -sign(f(xr))`` 
468      elementwise. 
469   
470      The function is guaranteed to find a valid bracket if the function is monotonic, 
471      but it may find a bracket under other conditions. 
472   
473      This function works elementwise when `xl0`, `xr0`, `xmin`, `xmax`, `factor`, and 
474      the elements of `args` are (mutually broadcastable) arrays. 
475   
476      Parameters 
477      ---------- 
478      f : callable 
479          The function for which the root is to be bracketed. The signature must be:: 
480   
481              f(x: array, *args) -> array 
482   
483          where each element of ``x`` is a finite real and ``args`` is a tuple, 
484          which may contain an arbitrary number of arrays that are broadcastable 
485          with ``x``. 
486   
487          `f` must be an elementwise function: each element ``f(x)[i]`` 
488          must equal ``f(x[i])`` for all indices ``i``. It must not mutate the 
489          array ``x`` or the arrays in ``args``. 
490      xl0, xr0: float array_like 
491          Starting guess of bracket, which need not contain a root. If `xr0` is 
492          not provided, ``xr0 = xl0 + 1``. Must be broadcastable with all other 
493          array inputs. 
494      xmin, xmax : float array_like, optional 
495          Minimum and maximum allowable endpoints of the bracket, inclusive. Must 
496          be broadcastable with all other array inputs. 
497      factor : float array_like, default: 2 
498          The factor used to grow the bracket. See Notes. 
499      args : tuple of array_like, optional 
500          Additional positional array arguments to be passed to `f`. 
501          If the callable for which the root is desired requires arguments that are 
502          not broadcastable with `x`, wrap that callable with `f` such that `f` 
503          accepts only `x` and broadcastable ``*args``. 
504      maxiter : int, default: 1000 
505          The maximum number of iterations of the algorithm to perform. 
506   
507      Returns 
508      ------- 
509      res : _RichResult 
510          An object similar to an instance of `scipy.optimize.OptimizeResult` with the 
511          following attributes. The descriptions are written as though the values will 
512          be scalars; however, if `f` returns an array, the outputs will be 
513          arrays of the same shape. 
514   
515          success : bool array 
516              ``True`` where the algorithm terminated successfully (status ``0``); 
517              ``False`` otherwise. 
518          status : int array 
519              An integer representing the exit status of the algorithm. 
520   
521              - ``0`` : The algorithm produced a valid bracket. 
522              - ``-1`` : The bracket expanded to the allowable limits without success. 
523              - ``-2`` : The maximum number of iterations was reached. 
524              - ``-3`` : A non-finite value was encountered. 
525              - ``-4`` : Iteration was terminated by `callback`. 
526              - ``-5``: The initial bracket does not satisfy`xmin <= xl0 < xr0 < xmax`. 
527               
528          bracket : 2-tuple of float arrays 
529              The lower and upper endpoints of the bracket, if the algorithm 
530              terminated successfully. 
531          f_bracket : 2-tuple of float arrays 
532              The values of `f` evaluated at the endpoints of ``res.bracket``, 
533              respectively. 
534          nfev : int array 
535              The number of abscissae at which `f` was evaluated to find the root. 
536              This is distinct from the number of times `f` is *called* because the 
537              the function may evaluated at multiple points in a single call. 
538          nit : int array 
539              The number of iterations of the algorithm that were performed. 
540   
541      Notes 
542      ----- 
543      This function generalizes an algorithm found in pieces throughout the 
544      `scipy.stats` codebase. The strategy is to iteratively grow the bracket `(l, r)` 
545      until ``f(l) < 0 < f(r)`` or ``f(r) < 0 < f(l)``. The bracket grows to the left 
546      as follows. 
547   
548      - If `xmin` is not provided, the distance between `xl0` and `l` is iteratively 
549        increased by `factor`. 
550      - If `xmin` is provided, the distance between `xmin` and `l` is iteratively 
551        decreased by `factor`. Note that this also *increases* the bracket size. 
552   
553      Growth of the bracket to the right is analogous. 
554   
555      Growth of the bracket in one direction stops when the endpoint is no longer 
556      finite, the function value at the endpoint is no longer finite, or the 
557      endpoint reaches its limiting value (`xmin` or `xmax`). Iteration terminates 
558      when the bracket stops growing in both directions, the bracket surrounds 
559      the root, or a root is found (by chance). 
560   
561      If two brackets are found - that is, a bracket is found on both sides in 
562      the same iteration, the smaller of the two is returned. 
563       
564      If roots of the function are found, both `xl` and `xr` are set to the 
565      leftmost root. 
566       
567      See Also 
568      -------- 
569      find_root 
570   
571      Examples 
572      -------- 
573      Suppose we wish to find the root of the following function. 
574   
575      >>> def f(x, c=5): 
576      ...     return x**3 - 2*x - c 
577   
578      First, we must find a valid bracket. The function is not monotonic, 
579      but `bracket_root` may be able to provide a bracket. 
580   
581      >>> from scipy.optimize import elementwise 
582      >>> res_bracket = elementwise.bracket_root(f, 0) 
583      >>> res_bracket.success 
584      True 
585      >>> res_bracket.bracket 
586      (2.0, 4.0) 
587   
588      Indeed, the values of the function at the bracket endpoints have 
589      opposite signs. 
590   
591      >>> res_bracket.f_bracket 
592      (-1.0, 51.0) 
593   
594      Once we have a valid bracket, `find_root` can be used to provide 
595      a precise root. 
596   
597      >>> res_root = elementwise.find_root(f, res_bracket.bracket) 
598      >>> res_root.x 
599      2.0945514815423265 
600   
601      `bracket_root` and `find_root` accept arrays for most arguments. 
602      For instance, to find the root for a few values of the parameter ``c`` 
603      at once: 
604   
605      >>> import numpy as np 
606      >>> c = np.asarray([3, 4, 5]) 
607      >>> res_bracket = elementwise.bracket_root(f, 0, args=(c,)) 
608      >>> res_bracket.bracket 
609      (array([1., 1., 2.]), array([2., 2., 4.])) 
610      >>> res_root = elementwise.find_root(f, res_bracket.bracket, args=(c,)) 
611      >>> res_root.x 
612      array([1.8932892 , 2.        , 2.09455148]) 
613   
614      """  # noqa: E501
615  
616      res = _bracket_root(f, xl0, xr0=xr0, xmin=xmin, xmax=xmax, factor=factor,
617                          args=args, maxiter=maxiter)
618      res.bracket = res.xl, res.xr
619      res.f_bracket = res.fl, res.fr
620      del res.xl
621      del res.xr
622      del res.fl
623      del res.fr
624      return res

627  def bracket_minimum(f, xm0, *, xl0=None, xr0=None, xmin=None, xmax=None,
628                       factor=None, args=(), maxiter=1000):
629      """Bracket the minimum of a unimodal, real-valued function of a real variable. 
630   
631      For each element of the output of `f`, `bracket_minimum` seeks the scalar 
632      bracket points ``xl < xm < xr`` such that ``fl >= fm <= fr`` where one of the 
633      inequalities is strict. 
634   
635      The function is guaranteed to find a valid bracket if the function is 
636      strongly unimodal, but it may find a bracket under other conditions. 
637   
638      This function works elementwise when `xm0`, `xl0`, `xr0`, `xmin`, `xmax`, `factor`, 
639      and the elements of `args` are (mutually broadcastable) arrays. 
640   
641      Parameters 
642      ---------- 
643      f : callable 
644          The function for which the root is to be bracketed. The signature must be:: 
645   
646              f(x: array, *args) -> array 
647   
648          where each element of ``x`` is a finite real and ``args`` is a tuple, 
649          which may contain an arbitrary number of arrays that are broadcastable 
650          with ``x``. 
651   
652          `f` must be an elementwise function: each element ``f(x)[i]`` 
653          must equal ``f(x[i])`` for all indices ``i``. It must not mutate the 
654          array ``x`` or the arrays in ``args``. 
655      xm0: float array_like 
656          Starting guess for middle point of bracket. 
657      xl0, xr0: float array_like, optional 
658          Starting guesses for left and right endpoints of the bracket. Must 
659          be broadcastable with all other array inputs. 
660      xmin, xmax : float array_like, optional 
661          Minimum and maximum allowable endpoints of the bracket, inclusive. Must 
662          be broadcastable with all other array inputs. 
663      factor : float array_like, default: 2 
664          The factor used to grow the bracket. See Notes. 
665      args : tuple of array_like, optional 
666          Additional positional array arguments to be passed to `f`. 
667          If the callable for which the root is desired requires arguments that are 
668          not broadcastable with `x`, wrap that callable with `f` such that `f` 
669          accepts only `x` and broadcastable ``*args``. 
670      maxiter : int, default: 1000 
671          The maximum number of iterations of the algorithm to perform. 
672   
673      Returns 
674      ------- 
675      res : _RichResult 
676          An object similar to an instance of `scipy.optimize.OptimizeResult` with the 
677          following attributes. The descriptions are written as though the values will 
678          be scalars; however, if `f` returns an array, the outputs will be 
679          arrays of the same shape. 
680   
681          success : bool array 
682              ``True`` where the algorithm terminated successfully (status ``0``); 
683              ``False`` otherwise. 
684          status : int array 
685              An integer representing the exit status of the algorithm. 
686   
687              - ``0`` : The algorithm produced a valid bracket. 
688              - ``-1`` : The bracket expanded to the allowable limits. Assuming 
689                unimodality, this implies the endpoint at the limit is a minimizer. 
690              - ``-2`` : The maximum number of iterations was reached. 
691              - ``-3`` : A non-finite value was encountered. 
692              - ``-4`` : ``None`` shall pass. 
693              - ``-5`` : The initial bracket does not satisfy 
694                `xmin <= xl0 < xm0 < xr0 <= xmax`. 
695   
696          bracket : 3-tuple of float arrays 
697              The left, middle, and right points of the bracket, if the algorithm 
698              terminated successfully. 
699          f_bracket : 3-tuple of float arrays 
700              The function value at the left, middle, and right points of the bracket. 
701          nfev : int array 
702              The number of abscissae at which `f` was evaluated to find the root. 
703              This is distinct from the number of times `f` is *called* because the 
704              the function may evaluated at multiple points in a single call. 
705          nit : int array 
706              The number of iterations of the algorithm that were performed. 
707   
708      Notes 
709      ----- 
710      Similar to `scipy.optimize.bracket`, this function seeks to find real 
711      points ``xl < xm < xr`` such that ``f(xl) >= f(xm)`` and ``f(xr) >= f(xm)``, 
712      where at least one of the inequalities is strict. Unlike `scipy.optimize.bracket`, 
713      this function can operate in a vectorized manner on array input, so long as 
714      the input arrays are broadcastable with each other. Also unlike 
715      `scipy.optimize.bracket`, users may specify minimum and maximum endpoints 
716      for the desired bracket. 
717   
718      Given an initial trio of points ``xl = xl0``, ``xm = xm0``, ``xr = xr0``, 
719      the algorithm checks if these points already give a valid bracket. If not, 
720      a new endpoint, ``w`` is chosen in the "downhill" direction, ``xm`` becomes the new 
721      opposite endpoint, and either `xl` or `xr` becomes the new middle point, 
722      depending on which direction is downhill. The algorithm repeats from here. 
723   
724      The new endpoint `w` is chosen differently depending on whether or not a 
725      boundary `xmin` or `xmax` has been set in the downhill direction. Without 
726      loss of generality, suppose the downhill direction is to the right, so that 
727      ``f(xl) > f(xm) > f(xr)``. If there is no boundary to the right, then `w` 
728      is chosen to be ``xr + factor * (xr - xm)`` where `factor` is controlled by 
729      the user (defaults to 2.0) so that step sizes increase in geometric proportion. 
730      If there is a boundary, `xmax` in this case, then `w` is chosen to be 
731      ``xmax - (xmax - xr)/factor``, with steps slowing to a stop at 
732      `xmax`. This cautious approach ensures that a minimum near but distinct from 
733      the boundary isn't missed while also detecting whether or not the `xmax` is 
734      a minimizer when `xmax` is reached after a finite number of steps. 
735   
736      See Also 
737      -------- 
738      scipy.optimize.bracket 
739      scipy.optimize.elementwise.find_minimum 
740   
741      Examples 
742      -------- 
743      Suppose we wish to minimize the following function. 
744   
745      >>> def f(x, c=1): 
746      ...     return (x - c)**2 + 2 
747   
748      First, we must find a valid bracket. The function is unimodal, 
749      so `bracket_minium` will easily find a bracket. 
750   
751      >>> from scipy.optimize import elementwise 
752      >>> res_bracket = elementwise.bracket_minimum(f, 0) 
753      >>> res_bracket.success 
754      True 
755      >>> res_bracket.bracket 
756      (0.0, 0.5, 1.5) 
757   
758      Indeed, the bracket points are ordered and the function value 
759      at the middle bracket point is less than at the surrounding 
760      points. 
761   
762      >>> xl, xm, xr = res_bracket.bracket 
763      >>> fl, fm, fr = res_bracket.f_bracket 
764      >>> (xl < xm < xr) and (fl > fm <= fr) 
765      True 
766   
767      Once we have a valid bracket, `find_minimum` can be used to provide 
768      an estimate of the minimizer. 
769   
770      >>> res_minimum = elementwise.find_minimum(f, res_bracket.bracket) 
771      >>> res_minimum.x 
772      1.0000000149011612 
773   
774      `bracket_minimum` and `find_minimum` accept arrays for most arguments. 
775      For instance, to find the minimizers and minima for a few values of the 
776      parameter ``c`` at once: 
777   
778      >>> import numpy as np 
779      >>> c = np.asarray([1, 1.5, 2]) 
780      >>> res_bracket = elementwise.bracket_minimum(f, 0, args=(c,)) 
781      >>> res_bracket.bracket 
782      (array([0. , 0.5, 0.5]), array([0.5, 1.5, 1.5]), array([1.5, 2.5, 2.5])) 
783      >>> res_minimum = elementwise.find_minimum(f, res_bracket.bracket, args=(c,)) 
784      >>> res_minimum.x 
785      array([1.00000001, 1.5       , 2.        ]) 
786      >>> res_minimum.f_x 
787      array([2., 2., 2.]) 
788   
789      """  # noqa: E501
790  
791      res = _bracket_minimum(f, xm0, xl0=xl0, xr0=xr0, xmin=xmin, xmax=xmax,
792                             factor=factor, args=args, maxiter=maxiter)
793      res.bracket = res.xl, res.xm, res.xr
794      res.f_bracket = res.fl, res.fm, res.fr
795      del res.xl
796      del res.xm
797      del res.xr
798      del res.fl
799      del res.fm
800      del res.fr
801      return res
